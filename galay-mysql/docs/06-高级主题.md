# 06-高级主题

本页聚焦当前源码已经暴露出来、但在快速开始中不展开的进阶主题。

## 超时与 buffer 调优

### 同步路径

同步客户端的超时入口是 `MysqlConfig::connect_timeout_ms`，它只覆盖连接阶段。

### 异步路径

异步客户端的超时入口是 `AsyncMysqlConfig`：

- `send_timeout`
- `recv_timeout`
- `buffer_size`
- `result_row_reserve_hint`

推荐策略：

- 默认先设置 `recv_timeout`
- 对大结果集场景适当提高 `buffer_size`
- 对已知结果行数的查询设置 `result_row_reserve_hint`

## `result_row_reserve_hint` 的使用时机

`AsyncMysqlConfig::result_row_reserve_hint` 是结果集行预分配提示：

- `0` 表示不预分配
- 已知返回行数上界时，可以减少多次扩容
- 返回行数波动很大时，不建议盲目设置过大的 hint

## `stmtExecute()` 的零拷贝参数层

异步 `stmtExecute()` 暴露了两套参数层：

```cpp
MysqlStmtExecuteAwaitable stmtExecute(uint32_t stmt_id,
                                      std::span<const std::optional<std::string>> params,
                                      std::span<const uint8_t> param_types = {});

MysqlStmtExecuteAwaitable stmtExecute(uint32_t stmt_id,
                                      std::span<const std::optional<std::string_view>> params,
                                      std::span<const uint8_t> param_types = {});
```

如果调用方已经持有稳定的字符串视图，可优先使用 `std::string_view` 版本，避免无意义复制。

## `pipeline()` 与 `batch()` 的取舍

- `pipeline()` 直接接收 `std::span<const std::string_view>`，适合业务层
- `batch()` 接收 `std::span<const protocol::MysqlCommandView>`，适合已经使用 `MysqlCommandBuilder` 的底层路径

如果没有特殊原因，优先 `pipeline()`；只有在你需要复用编码后的命令包时再进入 `batch()`。

## 连接池 ownership 与归还

`MysqlConnectionPool::acquire()` 成功后返回 `AsyncMysqlClient*`，但对象所有权不转移：

- 指针只能在连接池存活期间使用
- 使用完成后必须 `pool.release(client)`
- 业务层不应自行 `delete` 这个指针

## 模块 / import 现实约束

当前 import 构建条件来自 `cmake/option.cmake` 的自动判定：

- CMake 版本必须满足模块链路要求
- 生成器必须是 `Ninja` 或 `Visual Studio`
- GCC 路径当前只在 `Linux + GCC >= 14` 开启
- Clang 路径要求 `clang-scan-deps` 可用，且不是 `AppleClang`

因此：

- `GALAY_MYSQL_BUILD_MODULE_EXAMPLES=ON` 不保证一定生成 import 示例
- 文档中的 import 命令都应理解为“满足工具链条件时再执行”

## 认证与协议边界

根据当前源码：

- 认证代码显式处理 `mysql_native_password` 与 `caching_sha2_password`
- 同步与异步路径都支持 `caching_sha2_password` 的 fast auth，以及服务端返回 `0x04` 时的 RSA 公钥 full auth
- `protocol::AuthPlugin::cachingSha2FullAuth()` 提供公钥 full auth 加密辅助，`T2-MysqlAuth` 覆盖了这条 helper 路径
- 异步路径对 auth switch 返回错误：`Auth switch is not supported`
- 同步路径里的 `Full auth not supported` 仅对应未覆盖的其他 full-auth 变体，不适用于已经实现的 `caching_sha2_password` 公钥流程
- 文档不把传输层 TLS 当作已验证公开能力；OpenSSL 在当前仓库中主要用于认证相关加密辅助

## 关于 awaitable 语义的高级注意点

- awaitable 是按值返回的对象
- 业务代码应把它们当作一次性对象使用
- 当前实现与 tests/examples 的契约是：成功后 `expected` 成功且 `optional` 通常有值
- 空 `optional` 不是推荐的“继续等待”协议，而是需要被视为异常状态

## 生产环境注意事项

- 不要在同一个 `AsyncMysqlClient` 上叠加并发操作
- 为数据库侧连接数限制设置合理的 `max_connections`
- 在事务路径里显式处理提交与回滚
- benchmark 数字必须附原始输出，不能只留转抄表格
- 把 README / docs / 自动化验证都对齐到真实源码 target 名
