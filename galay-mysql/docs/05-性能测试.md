# 05-性能测试

本页只记录当前仓库真实存在的 benchmark 入口、参数与复现方式。

## 当前状态（2026-03-10）

- `benchmark/CMakeLists.txt` 只定义了两个 target：`B1-SyncPressure` 与 `B2-AsyncPressure`
- benchmark CLI 的公共参数来自 `benchmark/common/BenchmarkConfig.h`
- 本次文档修复 **未重新产出新的性能数值**；因此本页不再保留无法追溯到原始输出的 QPS / 延迟表格
- 如果你需要发布性能数字，请连同命令、环境、原始 stdout 一并存档

## 构建 benchmark

要生成 benchmark 二进制，必须启用 `GALAY_MYSQL_BUILD_BENCHMARKS`：

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_MYSQL_BUILD_BENCHMARKS=ON
cmake --build build --target B1-SyncPressure B2-AsyncPressure --parallel
```

## Rust 对照 benchmark

仓库包含最小 Rust 对照实现：

- `benchmark/compare/rust/src/bin/rust_sync_pressure.rs`
- `benchmark/compare/rust/src/bin/rust_async_pressure.rs`

统一对比入口：

```bash
GALAY_MYSQL_HOST=127.0.0.1 \
GALAY_MYSQL_PORT=3306 \
GALAY_MYSQL_USER=root \
GALAY_MYSQL_PASSWORD=password \
GALAY_MYSQL_DB=test \
./scripts/S2-Bench-Rust-Compare.sh ./build
```

脚本会在两个场景下输出 C++ / Rust 的 `qps`：

- `B1-SyncPressure` vs `rust_sync_pressure`
- `B2-AsyncPressure` vs `rust_async_pressure`

## 发布要求

- 发布任何 benchmark 结论前，必须跑 `scripts/S2-Bench-Rust-Compare.sh`，确保 C++ 与 Rust 在同一组参数下比对，本脚本会在摘要里打印 `start_time`/`end_time`（UTC）、吞吐和 p50/p95/p99 延迟，并附加资源指标仍需人工采集的说明。
- 将脚本原始 stdout 通过 `tee` 或重定向保存到带有时间戳的文件（例如 `benchmark-results/2026-04-12-S2-Bench-Rust-Compare.txt`），文件内就包含了命令、环境变量、开始与结束时间。该文件加上 start_time、commit 及测试日期便是公开数字的主要证据链。
- 资源占用（CPU、内存、网络等）目前还没有自动化采集，请在执行 benchmark 时用 `top`、`ps`、平台面板等手动截取快照，并把这些补充记录与 stdout/命令一起归档。

| target | 源码文件 | 说明 |

| `B2-AsyncPressure` | `benchmark/B2-async_pressure.cc` | 异步客户端压测 |

## 真实 CLI 参数

`BenchmarkConfig.h` 当前导出的 CLI 选项为：

- `--clients N`
- `--queries N`
- `--warmup N`
- `--timeout-sec N`
- `--sql "SELECT 1"`
- `--mode normal|batch|pipeline`
- `--batch-size N`
- `--buffer-size N`
- `--alloc-stats`

建议先查看帮助输出：

- benchmark 二进制当前没有专门的 `--help` 开关
- 参数列表以 `benchmark/common/BenchmarkConfig.h` 中的 `printUsage()` 输出为准
- 传入未知参数时，程序会打印 usage 并以非零状态退出

## 运行命令示例

### `B1-SyncPressure`

```bash
GALAY_MYSQL_HOST=127.0.0.1 \
GALAY_MYSQL_PORT=3306 \
GALAY_MYSQL_USER=root \
GALAY_MYSQL_PASSWORD=password \
GALAY_MYSQL_DB=test \
./build/benchmark/B1-SyncPressure \
  --clients 8 \
  --queries 1000 \
  --warmup 100 \
  --timeout-sec 180 \
  --sql "SELECT 1"
```

### `B2-AsyncPressure`

```bash
GALAY_MYSQL_HOST=127.0.0.1 \
GALAY_MYSQL_PORT=3306 \
GALAY_MYSQL_USER=root \
GALAY_MYSQL_PASSWORD=password \
GALAY_MYSQL_DB=test \

  --queries 200 \
```

## 建议记录的证据

每次 benchmark 至少记录以下信息：

1. 日期与 commit / worktree
2. 机器型号、CPU、内存、操作系统
3. 编译器与 CMake 版本
4. MySQL 版本与部署方式
5. 完整运行命令
6. benchmark 原始 stdout
7. 是否启用了 `--alloc-stats`

推荐做法：

```bash
mkdir -p benchmark-results
GALAY_MYSQL_HOST=127.0.0.1 \
GALAY_MYSQL_PORT=3306 \
GALAY_MYSQL_USER=root \
GALAY_MYSQL_PASSWORD=password \
GALAY_MYSQL_DB=test \
./build/benchmark/B2-AsyncPressure \
  --clients 64 \
  --queries 200 \
  --warmup 50 \
  --timeout-sec 180 \
  --mode pipeline \
  --batch-size 16 \
  --buffer-size 16384 \
  --sql "SELECT 1" | tee benchmark-results/2026-03-10-B2-AsyncPressure.txt
```

## 如何解释 benchmark 模式

- `normal`：按默认单条语句流程执行
- `batch`：使用批量命令模式
- `pipeline`：使用 pipeline 模式发送多条语句

具体语义仍以 `benchmark/common/BenchmarkConfig.h` 与相应 benchmark 源码为准。
