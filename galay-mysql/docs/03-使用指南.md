# 03-使用指南

本页面向仓库使用者，说明什么时候选同步、异步或连接池，以及如何把库接到外部项目中。

## 先选哪条接入路径

| 场景 | 推荐接口 |
| --- | --- |
| 简单 CLI、迁移脚本、初始化逻辑 | `MysqlClient` |
| 已经使用 `galay-kernel` runtime / scheduler | `AsyncMysqlClient` |
| 多协程复用连接、希望限制总连接数 | `MysqlConnectionPool` |

## 外部 CMake 项目接入

安装本仓库后，在外部项目中保持如下用法：

```cmake
find_package(GalayMysql REQUIRED CONFIG)

add_executable(app main.cc)
target_link_libraries(app PRIVATE galay-mysql::galay-mysql)
```

安装树中的主配置文件名是 `GalayMysqlConfig.cmake`，并会额外提供 `galay-mysqlConfig.cmake` 兼容入口；本文档统一使用 `GalayMysql` 作为包名。

如果只使用 include 路径，头文件入口通常是：

```cpp
#include "galay-mysql/async/AsyncMysqlClient.h"
#include "galay-mysql/async/MysqlConnectionPool.h"
#include "galay-mysql/sync/MysqlClient.h"
```

如果使用 import / module 路径，需要确保当前工具链满足 `cmake/option.cmake` 的检测条件。

## 同步客户端工作流

### 1. 建立连接

```cpp
MysqlConfig cfg = MysqlConfig::create("127.0.0.1", 3306, "root", "password", "test");
cfg.connect_timeout_ms = 5000;

MysqlClient client;
auto connect_result = client.connect(cfg);
if (!connect_result) {
    std::cerr << connect_result.error().message() << '\n';
    return;
}
```

### 2. 执行查询

```cpp
auto query_result = client.query("SELECT id, name FROM users LIMIT 5");
if (!query_result) {
    std::cerr << query_result.error().message() << '\n';
    client.close();
    return;
}

for (size_t i = 0; i < query_result->rowCount(); ++i) {
    const auto& row = query_result->row(i);
    std::cout << row.getString(0) << ' ' << row.getString(1) << '\n';
}
```

### 3. 预处理语句与事务

```cpp
auto prep = client.prepare("SELECT ? + ?");
if (!prep) {
    client.close();
    return;
}

client.beginTransaction();
std::vector<std::optional<std::string>> params{"3", "5"};
auto exec = client.stmtExecute(prep->statement_id, params);
if (exec && exec->rowCount() > 0) {
    std::cout << exec->row(0).getString(0) << '\n';
}
client.stmtClose(prep->statement_id);
client.commit();
```

## 异步客户端工作流

### 1. 创建 runtime / scheduler

`AsyncMysqlClient` 依赖 `galay-kernel` 的 `Runtime` / `IOScheduler`：

```cpp
Runtime runtime;
runtime.start();
auto* scheduler = runtime.getNextIOScheduler();
```

### 2. 创建客户端并连接

```cpp
auto client = AsyncMysqlClientBuilder()
    .scheduler(scheduler)
    .config(AsyncMysqlConfig::withRecvTimeout(std::chrono::seconds(5)))
    .build();

auto connect_result = co_await client.connect("127.0.0.1", 3306, "root", "password", "test");
if (!connect_result || !connect_result->has_value()) {
    co_return;
}
```

### 3. 查询、pipeline 与关闭

```cpp
auto query_result = co_await client.query("SELECT 1");
if (query_result && query_result->has_value()) {
    std::cout << query_result->value().row(0).getString(0) << '\n';
}

std::vector<std::string_view> sqls{"SELECT 11", "SELECT 22", "SELECT 33"};
auto pipeline_result = co_await client.pipeline(sqls);
if (pipeline_result && pipeline_result->has_value()) {
    for (const auto& rs : pipeline_result->value()) {
        std::cout << rs.row(0).getString(0) << '\n';
    }
}

co_await client.close();
```

### 4. awaitable 使用规则

- 公开接口返回 awaitable 值对象，而不是引用
- 每个 awaitable 按仓库当前示例与测试的方式执行一次 `co_await`
- 成功后继续检查一次 `has_value()`，把空 `optional` 当作异常状态处理

## 连接池工作流

### 1. 配置连接池

```cpp
MysqlConnectionPoolConfig pool_cfg;
pool_cfg.mysql_config = MysqlConfig::create("127.0.0.1", 3306, "root", "password", "test");
pool_cfg.async_config = AsyncMysqlConfig::noTimeout();
pool_cfg.min_connections = 2;
pool_cfg.max_connections = 8;

MysqlConnectionPool pool(scheduler, pool_cfg);
```

### 2. 获取与归还连接

```cpp
auto acquire_result = co_await pool.acquire();
if (!acquire_result || !acquire_result->has_value()) {
    co_return;
}

AsyncMysqlClient* client = acquire_result->value();
auto query_result = co_await client->query("SELECT 1");
if (query_result && query_result->has_value()) {
    std::cout << query_result->value().row(0).getString(0) << '\n';
}

pool.release(client);
```

### 3. 连接池注意事项

- 成功 `acquire()` 后必须 `release(client)`
- 归还遗漏会导致池内可用连接减少
- 池返回的是裸指针，但对象所有权仍在池内

## `batch()` 与 `pipeline()` 如何选

| 接口 | 输入类型 | 适用场景 |
| --- | --- | --- |
| `pipeline()` | `std::span<const std::string_view>` | 已有多条 SQL 字符串，希望快速串联发送 |
| `batch()` | `std::span<const protocol::MysqlCommandView>` | 需要更底层地复用已经编码的命令视图 |

大多数业务代码优先使用 `pipeline()`；只有在你已经基于 `MysqlCommandBuilder` 组装命令时再选 `batch()`。

## 结果集与错误处理建议

- `SELECT` 结果优先看 `rowCount()`、`fieldCount()`、`row(i)`
- `INSERT / UPDATE / DELETE` 结果优先看 `affectedRows()`、`lastInsertId()`
- 统一把 `MysqlError::message()` 作为日志首要输出
- 同步与异步路径都应保留 MySQL 服务端错误码 `serverErrno()`

## 生产接入清单

- 为同步路径设置 `connect_timeout_ms`
- 为异步路径明确设置 `send_timeout` / `recv_timeout` / `buffer_size`
- 在事务与预处理语句场景里显式关闭 / 提交 / 回滚
- 对 benchmark 结果保留原始 stdout，而不是只复制表格数字
- 发布 benchmark 结论前必须使用 `scripts/S2-Bench-Rust-Compare.sh` 跑 C++/Rust 同参对比，该脚本会在输出中打印 `start_time`/`end_time`（UTC）、吞吐及 p50/p95/p99 延迟，同时备注资源量化仍需人工；请用带时间戳的文件名（例如 `benchmark-results/2026-04-12-S2-Bench-Rust-Compare.txt`）保存 stdout，以保留命令、环境变量和时间信息。
- 资源占用目前还没有自动化采集，需要额外手动记录 CPU/内存等指标并与 benchmark 输出一起存档。
- 把示例命令与 target 名直接绑定到真实文件名，不自创别名
