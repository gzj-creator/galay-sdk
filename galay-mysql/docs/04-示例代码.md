# 04-示例代码

本页只记录仓库内真实存在、可由 `examples/CMakeLists.txt` 生成 target 的示例，不再使用与源码不一致的 CamelCase 别名。本页同时取代旧的 `examples/README.md`，作为唯一示例索引入口。

## 构建前提

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_MYSQL_BUILD_EXAMPLES=ON
cmake --build build --parallel
```

import 示例需要额外满足模块工具链条件：

下面给的是可直接复制的 `Ninja` 版本；如果你使用的是 Visual Studio 生成器，请仅替换 `-G` 参数。

```bash
cmake -S . -B build-import -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_MYSQL_BUILD_EXAMPLES=ON \
  -DGALAY_MYSQL_ENABLE_IMPORT_COMPILATION=ON \
  -DGALAY_MYSQL_BUILD_MODULE_EXAMPLES=ON
cmake --build build-import --parallel
```

## 运行前环境变量

示例公共配置来自 `examples/common/ExampleConfig.h`，优先读取：

- `GALAY_MYSQL_HOST`
- `GALAY_MYSQL_PORT`
- `GALAY_MYSQL_USER`
- `GALAY_MYSQL_PASSWORD`
- `GALAY_MYSQL_DB`

兼容回退变量：

- `MYSQL_HOST`
- `MYSQL_PORT`
- `MYSQL_USER`
- `MYSQL_PASSWORD`
- `MYSQL_DATABASE`

## 每个示例验证的能力

### `E1-async_query`

- 创建 `Runtime` / `IOScheduler`
- 使用 `AsyncMysqlClientBuilder` 构造客户端
- `co_await client.connect(...)`
- `co_await client.query("SELECT 1")`
- `co_await client.close()`

### `E2-sync_query`

- 使用 `MysqlConfig::create(...)`
- `MysqlClient::connect()`
- `MysqlClient::query()`
- 遍历 `MysqlResultSet`

### `E3-async_pool`

- 配置 `MysqlConnectionPoolConfig`
- `co_await pool.acquire()`
- 对 `AsyncMysqlClient*` 执行查询
- `pool.release(client)`

### `E4-sync_prepared_tx`

- `beginTransaction()`
- `prepare()`
- `stmtExecute()`
- `stmtClose()`
- `commit()`

### `E5-async_pipeline`

- 构造 `std::vector<std::string_view>`
- `co_await client.pipeline(...)`
- 校验返回结果个数与内容

## 推荐验证顺序

1. 先跑 `E2-sync_query-Include`，确认数据库连通性
2. 再跑 `E1-async_query-Include` 和 `E3-async_pool-Include`
3. 需要事务与预处理时跑 `E4-sync_prepared_tx-Include`
4. 需要串联查询时跑 `E5-async_pipeline-Include`
5. import / module 版本只在工具链确认可用后再跑
