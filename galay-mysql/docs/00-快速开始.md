# 00-快速开始

本页给出从源码构建、安装、运行 tests/examples/benchmarks 到外部项目 `find_package` 接入的最短路径。

## 构建前提

| 场景 | 真实要求 |
| --- | --- |
| 基础 include 构建 | CMake `>= 3.20`、可用的 C++23 / `std::expected` 工具链、`galay-kernel`、OpenSSL、spdlog |
| import / module 构建 | CMake `>= 3.28`、`Ninja` 或 `Visual Studio`、`GALAY_MYSQL_ENABLE_IMPORT_COMPILATION=ON` |
| GCC import 路径 | `Linux + GCC >= 14` |
| Clang import 路径 | 非 `AppleClang` 且可找到 `clang-scan-deps` |

运行 examples/tests/benchmarks 还需要一个可访问的 MySQL 服务端。

## 获取源码并构建

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_TESTING=ON \
  -DGALAY_MYSQL_BUILD_EXAMPLES=ON \
  -DGALAY_MYSQL_BUILD_BENCHMARKS=ON
cmake --build build --parallel
```

如果你只需要库本身，可以按需关闭测试、示例或 benchmark：

- `BUILD_TESTING=OFF`
- `GALAY_MYSQL_BUILD_EXAMPLES=OFF`
- `GALAY_MYSQL_BUILD_BENCHMARKS=OFF`

## 关键 CMake 开关

- `BUILD_TESTING`：构建并注册 `test/` 下 CTest 项
- `GALAY_MYSQL_BUILD_TESTS`：兼容别名（已废弃，建议改用 `BUILD_TESTING`）
- `GALAY_MYSQL_BUILD_EXAMPLES`：构建 `examples/` 下 target
- `GALAY_MYSQL_BUILD_BENCHMARKS`：构建 `benchmark/` 下 target
- `GALAY_MYSQL_BUILD_SHARED_LIBS`：切换共享 / 静态库
- `GALAY_MYSQL_INSTALL_MODULE_INTERFACE`：安装 `.cppm` 模块接口文件
- `GALAY_MYSQL_ENABLE_IMPORT_COMPILATION`：允许 CMake 检测 import 编译链路
- `GALAY_MYSQL_BUILD_MODULE_EXAMPLES`：在 import 编译可用时构建 import 示例
- `GALAY_MYSQL_CXX_STANDARD`：默认 `23`，当前文档与示例按该默认值描述

兼容开关 `GALAY_MYSQL_BUILD_IMPORT_EXAMPLES` 仍会映射到 `GALAY_MYSQL_BUILD_MODULE_EXAMPLES`，但已废弃。

## 安装并在外部项目中使用

先安装：

```bash
cmake --install build --prefix /tmp/galay-mysql-install
```

然后在外部 `CMakeLists.txt` 中使用：

```cmake
find_package(galay-mysql REQUIRED CONFIG)

add_executable(app main.cc)
target_link_libraries(app PRIVATE galay-mysql::galay-mysql)
```

安装目录里的真实配置文件名是 `galay-mysql-config.cmake`；下游文档与示例统一使用 `galay-mysql`。

include 头文件入口：

```cpp
#include "galay-mysql/async/AsyncMysqlClient.h"
#include "galay-mysql/async/MysqlConnectionPool.h"
#include "galay-mysql/sync/MysqlClient.h"
```

模块入口文件是 `galay-mysql/module/galay.mysql.cppm`；只有工具链满足条件时，import 构建路径才会被启用，并额外生成 `galay-mysql-modules` 门面 target。
模块入口文件是 `galay-mysql/module/galay.mysql.cppm`；只有工具链满足条件时，import 构建路径才会被启用，并额外生成 `galay-mysql-modules` 门面 target。

## 配置 MySQL 环境变量

examples、tests、benchmarks 统一优先读取这些变量：

- `GALAY_MYSQL_HOST`
- `GALAY_MYSQL_PORT`
- `GALAY_MYSQL_USER`
- `GALAY_MYSQL_PASSWORD`
- `GALAY_MYSQL_DB`

`examples/common/ExampleConfig.h` 和 `benchmark/common/BenchmarkConfig.h` 还接受 `MYSQL_*` / `MYSQL_DATABASE` 兼容变量。

`test/TestMysqlConfig.h` 不再携带默认主机或凭据；`T3`-`T7` 缺少必要环境变量时会返回 skip code `125`，供 `ctest` 识别为 skipped。

## 异步最小示例

```cpp
#include <chrono>
#include <iostream>
#include <galay-kernel/kernel/Runtime.h>
#include "galay-mysql/async/AsyncMysqlClient.h"

using namespace std::chrono_literals;
using namespace galay::kernel;
using namespace galay::mysql;

Coroutine run(IOScheduler* scheduler)
{
    auto client = AsyncMysqlClientBuilder()
        .scheduler(scheduler)
        .config(AsyncMysqlConfig::withRecvTimeout(5s))
        .build();

    auto connect_result = co_await client.connect("127.0.0.1", 3306, "root", "password", "test");
    if (!connect_result || !connect_result->has_value()) {
        co_return;
    }

    auto query_result = co_await client.query("SELECT 1");
    if (!query_result || !query_result->has_value()) {
        co_await client.close();
        co_return;
    }

    std::cout << query_result->value().row(0).getString(0) << '\n';
    co_await client.close();
}
```

关键点：

- awaitable 由接口按值返回，不要绑定到 `auto&`
- 当前实现与测试都按“一次 `co_await` 一个 awaitable”使用
- 结果值类型是 `std::expected<std::optional<T>, MysqlError>`，成功后仍建议检查一次 `has_value()`

## 同步最小示例

```cpp
#include <iostream>
#include "galay-mysql/sync/MysqlClient.h"

using namespace galay::mysql;

int main()
{
    MysqlConfig cfg = MysqlConfig::create("127.0.0.1", 3306, "root", "password", "test");
    cfg.connect_timeout_ms = 5000;

    MysqlClient client;
    auto connect_result = client.connect(cfg);
    if (!connect_result) {
        std::cerr << connect_result.error().message() << '\n';
        return 1;
    }

    auto query_result = client.query("SELECT 1");
    if (!query_result) {
        std::cerr << query_result.error().message() << '\n';
        client.close();
        return 1;
    }

    std::cout << query_result->row(0).getString(0) << '\n';
    client.close();
    return 0;
}
```

## 已知限制

- import 路径的启用由 CMake 自动判定，不满足工具链条件时不会生成 import 示例。
- 传输层 TLS 未作为当前仓库的已验证公开能力文档化；OpenSSL 在源码中用于认证相关加密辅助。
- benchmark 页面现只记录真实 target、参数与复现流程；旧的未留证性能数值已从文档中移除。
