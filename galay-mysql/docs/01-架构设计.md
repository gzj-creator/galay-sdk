# 01-架构设计

本页说明当前仓库源码的真实分层方式，以及 `AsyncMysqlClient` / `MysqlClient` / `MysqlConnectionPool` 的职责边界。

## 目录与模块职责

| 目录 | 作用 |
| --- | --- |
| `galay-mysql/base/` | 连接配置、错误类型、结果集与字段 / 行抽象 |
| `galay-mysql/protocol/` | MySQL 协议编码、解析、认证与命令构建 |
| `galay-mysql/async/` | 异步客户端、awaitable、buffer provider、连接池 |
| `galay-mysql/sync/` | 同步阻塞客户端 |
| `galay-mysql/module/` | C++ module 接口与导出前导头 |
| `examples/` | include / import 两套真实示例 |
| `test/` | 针对协议、同步、异步、连接池、事务与预处理语句的真实验证入口 |
| `benchmark/` | `B1-SyncPressure` 与 `B2-AsyncPressure` 两个 benchmark |

## 文档真相来源

仓库文档按以下优先级维护：

1. 公开头文件与安装导出 target
2. 实现行为
3. `examples/`
4. `test/`
5. `benchmark/`
6. Markdown 文档

## 异步执行模型

### `AsyncMysqlClient` 是单连接状态机

`AsyncMysqlClient` 持有：

- 一个底层 `TcpSocket`
- 一个 MySQL parser / encoder
- 一个 ring buffer
- 一个 `AsyncMysqlConfig`

因此，最安全的使用方式是把它当作“单条请求 / 响应流”的顺序执行对象。

### awaitable 由接口按值返回

`connect()`、`query()`、`prepare()`、`stmtExecute()`、`pipeline()`、`batch()` 都返回 awaitable 值对象，而不是引用或协程句柄。当前 examples/tests 的真实用法是：

1. 创建局部 awaitable 值
2. 执行一次 `co_await`
3. 检查 `std::expected`
4. 检查内层 `optional` 是否有值

当前实现如果在成功路径上得到空 `optional`，会把它视为异常状态，而不是“还没完成、需要继续 await”的正常协议。

### 主要 awaitable 类型

| 接口 | 真实 awaitable 类型 | 成功结果 |
| --- | --- | --- |
| `connect()` | `MysqlConnectAwaitable` | `std::optional<bool>` |
| `query()` | `MysqlQueryAwaitable` | `std::optional<MysqlResultSet>` |
| `prepare()` | `MysqlPrepareAwaitable` | `std::optional<PrepareResult>` |
| `stmtExecute()` | `MysqlStmtExecuteAwaitable` | `std::optional<MysqlResultSet>` |
| `pipeline()` / `batch()` | `MysqlPipelineAwaitable` | `std::optional<std::vector<MysqlResultSet>>` |

### timeout 与 buffer

异步路径的超时与 buffer 来自 `AsyncMysqlConfig`：

- `send_timeout`
- `recv_timeout`
- `buffer_size`
- `result_row_reserve_hint`

这几个字段都直接影响 `AsyncMysqlClient` 内部 I/O 以及结果集构建行为。

## 同步执行模型

`MysqlClient` 是一个同步阻塞客户端，核心特点：

- 连接、查询、预处理、事务都在调用线程内完成
- 返回类型统一是 `std::expected<... , MysqlError>`
- `close()` 为同步 `void` 接口

同步路径适合：

- CLI / 工具型程序
- 启动阶段初始化脚本
- 不想引入 runtime / scheduler 的简单接入场景

## 连接池模型

`MysqlConnectionPool` 仅服务于异步客户端：

- `acquire()` 返回 `AcquireAwaitable`
- 成功时拿到 `AsyncMysqlClient*`
- 使用完成后必须 `release(client)` 归还给池

池配置来自 `MysqlConnectionPoolConfig`：

- `mysql_config`
- `async_config`
- `min_connections`
- `max_connections`

根据源码实现，池内部维护 idle 队列、waiter 队列与总连接计数；等待者在连接归还时被 resume。

## 结果集与错误模型

### 结果集模型

`MysqlResultSet` 同时承载：

- 字段定义 `MysqlField`
- 行数据 `MysqlRow`
- OK 包元数据（影响行数、insert id、warnings、status flags、info）

因此：

- `SELECT` 等返回列信息的语句通常 `hasResultSet() == true`
- `INSERT / UPDATE / DELETE` 等语句通常主要使用 `affectedRows()`、`lastInsertId()` 等元数据

### 错误模型

同步与异步接口统一使用 `MysqlError`，错误类别来自 `MysqlErrorType`，包括：

- 连接 / 认证 / 查询 / 协议错误
- 超时、发送、接收错误
- 连接关闭、预处理语句、事务、内部错误

## 模块与安装导出

安装导出来自 `galay-mysql/CMakeLists.txt`：

- `find_package(galay-mysql REQUIRED CONFIG)`
- 导出 target：`galay-mysql::galay-mysql`
- 模块接口文件：`galay-mysql/module/galay.mysql.cppm`

只有当 CMake 认定 import 编译链路可用时，库 target 才会把 module file set 作为真实编译输入；否则只安装 `.cppm` 文件供下游查看或手工处理。

## 当前设计边界

- 文档不把传输层 TLS 当作已验证公开能力；源码中的 OpenSSL 主要用于认证相关加密辅助。
- `benchmark/` 只定义两个压测入口；性能页以复现命令与证据记录为主，而不是无来源数字表。
- 对同一个 `AsyncMysqlClient` 叠加并发操作没有被 examples/tests 作为推荐模式验证。
