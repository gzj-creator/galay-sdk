# 03-使用指南

本页解决三个最容易漂移的问题：协议支持边界、TLS 调用顺序、h2 与 h2c 的真实差异。

## 支持矩阵

| 场景 | 公开入口 | 构建条件 | 真实示例 | 真实测试 / benchmark | 备注 |
| --- | --- | --- | --- | --- | --- |
| HTTP/1.1 server | `HttpServer` | 默认启用 | `E1-EchoServer`、`E11-StaticServer`、`E12-HttpProxy` | `T5-http_server`、`T14-static_file_transfer_modes`、`T15-range_etag_server`、`B1-HttpServer` | 标准明文能力。 |
| HTTP/1.1 client | `HttpClient` | 默认启用 | `E2-EchoClient` | `T6-http_client_awaitable`、`T7-http_client_awaitable_edge_cases`、`T16-http_client_timeout`、`B2-HttpClient` | 仅支持 `http://`。 |
| HTTPS | `HttpsServer`、`HttpsClient` | `-DGALAY_HTTP_ENABLE_SSL=ON` | `E5-HttpsServer`、`E6-HttpsClient` | `T21-https_server`、`T22-https_client`、`T23-https_stress_test`、`T24-simple_https_test`、`B14-HttpsServer` | 客户端需显式 `handshake()`。 |
| WebSocket (`ws`) | `WsClient` + `WsSession`；服务端通过 HTTP Upgrade 组合 | 默认启用 | `E3-WebsocketServer`、`E4-WebsocketClient` | `T18-ws_server`、`T19-ws_client`、`T20-websocket_client`、`B5-WebsocketServer`、`B6-WebsocketClient` | 无独立 `WsServer` 公共类。 |
| WSS client | `WssClient` + `WssSession` | `-DGALAY_HTTP_ENABLE_SSL=ON` | `E8-WssClient` | `B8-WssClient` | `connect()` 后必须 `handshake()`。 |
| WSS server pattern | `HttpsServer` + `WsUpgrade` + 手动帧处理 | `-DGALAY_HTTP_ENABLE_SSL=ON` | `E7-WssServer` | `B7-WssServer` | 示例级方案；当前仓库没有独立 `WssServer` 公共类。 |
| h2c | `H2cServer`、`H2cClient` | 默认启用 | `E9-H2cEchoServer`、`E10-H2cEchoClient` | `T26-h2c_server`、`T25-h2c_client`、`T43-h2c_client_shutdown`、`T51-h2c_server_fast_path`、`B3-H2cServer`、`B4-H2cClient`、`B10-H2cMuxServer`、`B11-H2cMuxClient` | 客户端必须 `upgrade()`。 |
| h2 | `H2Server`、`H2Client` | `-DGALAY_HTTP_ENABLE_SSL=ON` | `E13-H2EchoServer`、`E14-H2EchoClient` | `T28-h2_server`、`T29-h2_client`、`T30-h2_error_model` 至 `T53-h2_stream_pool`、`B12-H2Server`、`B13-H2Client` | `H2Client::connect()` 内部包含握手与 ALPN。 |

## 构建模式

### 明文功能

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=ON \
  -DBUILD_BENCHMARKS=ON \
  -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build --parallel
```

### TLS 功能

```bash
cmake -S . -B build-ssl \
  -DCMAKE_BUILD_TYPE=Release \
  -DGALAY_HTTP_ENABLE_SSL=ON \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=ON \
  -DBUILD_BENCHMARKS=ON \
  -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build-ssl --parallel
```

### 模块目标

```bash
cmake -S . -B build-mod -G Ninja \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_MODULE_EXAMPLES=ON \
  -DGALAY_HTTP_ENABLE_SSL=OFF
cmake --build build-mod --target galay-http-modules --parallel
```

注意：

- `BUILD_MODULE_EXAMPLES` 对应真实 target 是 `galay-http-modules`
- 根 `CMakeLists.txt` 会在 CMake `< 3.28`、非 `Ninja/Visual Studio` 生成器或 `AppleClang` 环境下自动关闭模块目标
- 当模块目标真实开启时，`examples/import/*.cpp` 与 `T59-module_smoke` 也会一起进入构建图
- 如需 TLS 相关 import 示例（`E5`~`E8`、`E13`、`E14`），把同样的构建改成 `-DGALAY_HTTP_ENABLE_SSL=ON`

## 原生验证入口

仓库当前的原生验证入口分两层：

- 精确 target 构建：`cmake --build build --target T1-http_parser --parallel 4`
- `CTest`：`ctest -N --test-dir build` 与 `ctest --output-on-failure --test-dir build -R '^T1-http_parser$'`

当模块门禁满足时，还会额外出现 `T59-module_smoke`，用于验证 `import galay.http;` / `import galay.websocket;` / `import galay.http2;` 的真实 consumer 路径；TLS 构建下它还会额外触达 `Https*` / `Wss*` / `H2*` builder。

## 真实调用序列

### HTTP / HTTPS

`HttpClient`：

```cpp
HttpClient client = HttpClientBuilder().build();
co_await client.connect("http://127.0.0.1:8080/");
auto session = client.getSession();
```

`HttpsClient`：

```cpp
HttpsClient client(HttpsClientBuilder()
    .verifyPeer(false)
    .build());

co_await client.connect("https://127.0.0.1:8443/");
co_await client.handshake();
auto session = client.getSession();
```

区别在于：当前公开头没有把 HTTPS 握手折叠进 `HttpsClient::connect()`。

### WS / WSS

`ws://`：

```cpp
WsClient client = WsClientBuilder().build();
co_await client.connect("ws://127.0.0.1:8080/ws");
auto session = client.getSession(WsWriterSetting::byClient());
auto upgrader = session.upgrade();
co_await upgrader();
```

`wss://`：

```cpp
WssClient client(WssClientBuilder()
    .verifyPeer(false)
    .build());

co_await client.connect("wss://127.0.0.1:9443/ws");
co_await client.handshake();
auto session = client.getSession(WsWriterSetting::byClient());
auto upgrader = session.upgrade();
co_await upgrader();
```

### h2c / h2

`h2c`：

```cpp
auto client = H2cClientBuilder().build();
co_await client.connect("127.0.0.1", 9080);
co_await client.upgrade("/");
auto stream = client.post("/echo", "hello", "text/plain");
co_await client.shutdown();
```

`h2`：

```cpp
H2Client client(H2ClientBuilder()
    .verifyPeer(false)
    .build());

auto connected = co_await client.connect("127.0.0.1", 9443);
if (connected) {
    auto stream = client.post("/echo", "hello", "text/plain");
    co_await client.close();
}
```

区别在于：

- `h2c` 依赖明文 Upgrade
- `h2` 依赖 TLS + ALPN=`h2`
- `H2Client::connect()` 内部已经完成 preface/settings 发送

## 证书与本地验证

仓库自带测试证书：

- `test/test.crt`
- `test/test.key`

本地自签名验证建议：

- 示例与 smoke test 使用 `verifyPeer(false)`，参考 `examples/include/E6-https_client.cpp`、`examples/include/E8-wss_client.cpp`、`examples/include/E14-h2_echo_client.cpp`
- 如需开启证书校验，使用 `caPath(...)` 并开启 `verifyPeer(true)`

## 已知限制

### WSS 服务端不是独立高层 API

`examples/include/E7-wss_server.cpp` 文件头已经明确声明：

- 当前 `SslSocket` 缺少 `readv`
- 因此 WSS 服务端示例采用 `HttpsServer` + 手动 WebSocket 帧处理
- 这意味着当前仓库支持 WSS 客户端公共 API，但服务端仍主要以“组合模式 / 示例模式”暴露

### 模块目标有额外条件

- CMake `< 3.28` 时根 `CMakeLists.txt` 会关闭 `BUILD_MODULE_EXAMPLES`
- 非 `Ninja` / `Visual Studio` 生成器时会关闭模块目标
- `AppleClang` 环境会关闭模块目标
- 当前仓库的 import 示例集中在 `examples/import/`，并在模块门禁满足时与 `include/` 树保持 E1~E14 场景对齐；其中 TLS 相关 target 还需要 `-DGALAY_HTTP_ENABLE_SSL=ON`

### benchmark 结果的“当前性”

- `benchmark/*.cc` 和 `benchmark/CMakeLists.txt` 才是 benchmark target 名与命令的真相来源
- 除非重新执行 benchmark，否则历史吞吐数字都应视为“历史记录”，不能当作当前环境保证值
