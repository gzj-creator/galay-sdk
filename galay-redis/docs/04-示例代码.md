# 04-示例代码

本页只列出能在当前仓库中真实定位到的示例、测试和运行命令。每一项都给出：

- 源文件路径
- CMake 目标名
- 推荐运行命令
- 需要的环境变量或外部基础设施

下文统一用 `<build-dir>` 表示你的 CMake binary dir；它可以是 `build`、`build-release` 或其他自定义目录。

## 标准 CTest 入口

`test/CMakeLists.txt` 现在把这些目标注册进了标准 CTest 矩阵，并按依赖打上标签：

- `unit`：纯本地离线测试，如 `T3`、`T8`、`T9`、`T10`、`T12`、`T14`、`T15`、`T16`、`T20`、`T24-interface-contract`
- `redis`：Redis 仓库的常规测试标签，既包含离线单测，也包含依赖本地 `127.0.0.1:6379` 的网络测试
- `integration`：真实 Cluster + Sentinel 集成测试；若未设置 `GALAY_IT_ENABLE=1`，会按 `[SKIP]` 语义跳过

常用命令：

```bash
ctest --test-dir <build-dir> --output-on-failure -L unit
ctest --test-dir <build-dir> --output-on-failure -L redis
GALAY_IT_ENABLE=1 ctest --test-dir <build-dir> --output-on-failure -L integration
```

## `T13-integration_cluster_sentinel` 环境要求

真实集成测试的入口有两个：

| 入口 | 命令 | 说明 |
|---|---|---|
| 直接运行目标 | `./<build-dir>/test/T13-integration_cluster_sentinel` | 需要手动准备 cluster + sentinel 环境 |
| 使用辅助脚本 | `test/integration/run_cluster_sentinel_integration.sh --build-dir <build-dir>` | 自动拉起 Docker Compose，构建目标，并设置环境变量 |

辅助脚本支持：

- `--build-dir <build-dir>` 或 `GALAY_BUILD_DIR=<build-dir>`
- `--binary <path>` 或 `GALAY_IT_BINARY=<path>`（直接指定现成二进制）
- 未显式传参时，优先尝试仓库根目录下的 `build`，再扫描已有的 `build*` 目录

脚本与测试会读取这些环境变量：

- `GALAY_IT_ENABLE`
- `GALAY_IT_CLUSTER_HOST`
- `GALAY_IT_CLUSTER_PORT`
- `GALAY_IT_MOVED_KEY`
- `GALAY_IT_ASK_KEY`
- `GALAY_IT_SENTINEL_HOST`
- `GALAY_IT_SENTINEL_PORT`
- `GALAY_IT_SENTINEL_MASTER_NAME`
- `GALAY_IT_SENTINEL_KEY`
- `GALAY_IT_TRIGGER_SENTINEL_FAILOVER`

## 如何选择事实来源

如果你要给团队或 AI 系统提供示例检索，建议优先级如下：

1. `examples/include/E1-async_basic_demo.cc`
2. `examples/include/E2-pipeline_demo.cc`
3. `test/T5-redis_client_timeout.cc`
4. `test/T10-redis_raw_command_api.cc`
5. `test/T11-topology_and_pubsub.cc`
6. `test/T13-integration_cluster_sentinel.cc`
7. `test/T15-awaitable_surface.cc`（当你要检索 awaitable / pool public surface 时，优先看它）

如果你要验证模块消费路径，则把对应的 `examples/import/E1`~`E3` 放到与 include 示例同级参考。

原因：

- 它们覆盖了当前 async 主路径
- 文件名和目标名一一对应
- 命令路径可直接落到 `examples/CMakeLists.txt` / `test/CMakeLists.txt`
