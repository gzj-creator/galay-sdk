# 03-使用指南

本页聚焦“怎么用当前仓库里真实存在、并且已有 examples/tests 证明的接口”。旧 async 写法已经统一迁移到 `RedisCommandBuilder` + `command` / `batch` / `execute`。

下文统一用 `<build-dir>` 表示你的 CMake binary dir；它可以是 `build`、`build-release` 或其他自定义目录。

如果模块工具链可用，`examples/import/` 里还提供了与 `examples/include/` 对齐的 `E1`~`E3` import 版本；业务语义相同，只是消费方式改为 `import galay.redis;`。

## 从旧 async 文档迁移

| 旧写法 | 当前写法 | 真实来源 |
|---|---|---|
| `co_await client.get("k")` | `co_await client.command(builder.get("k"))` | `examples/include/E1-async_basic_demo.cc` |
| `co_await client.set("k", "v")` | `co_await client.command(builder.set("k", "v"))` | `examples/include/E1-async_basic_demo.cc` |
| `co_await client.pipeline(commands)` | `co_await client.batch(builder.commands())` | `examples/include/E2-pipeline_demo.cc` |
| `cluster.executeByKeyAuto(key, cmd, args)` | `cluster.execute(cmd, args, key, true)` | `test/T13-integration_cluster_sentinel.cc` |
| `sub.subscribe("channel")` | `subscriber.command(builder.subscribe("channel"))` | `examples/include/E3-topology_pubsub_demo.cc` |

## 单命令执行

来源：`examples/include/E1-async_basic_demo.cc`

- 目标：`E1-async_basic_demo`
- 运行：`./<build-dir>/examples/E1-async_basic_demo 127.0.0.1 6379`
- 环境变量：无

```cpp
auto client = RedisClientBuilder().scheduler(scheduler).build();
RedisCommandBuilder builder;

auto connect_result = co_await client.connect(host, port).timeout(std::chrono::seconds(5));
auto set_result = co_await client.command(builder.set(key, value)).timeout(std::chrono::seconds(5));
auto get_result = co_await client.command(builder.get(key)).timeout(std::chrono::seconds(5));
```

建议的取值方式：

```cpp
if (get_result && get_result.value()) {
    const auto& values = get_result.value().value();
    if (!values.empty() && values[0].isString()) {
        std::cout << values[0].toString() << '\n';
    }
}
```

## 批量发送 / Pipeline

来源：`examples/include/E2-pipeline_demo.cc`、`test/T8-redis_batch_timeout_api.cc`、`test/T9-redis_batch_span_api.cc`

- 目标：`E2-pipeline_demo`
- 运行：`./<build-dir>/examples/E2-pipeline_demo 127.0.0.1 6379 demo:pipeline: 20`
- 环境变量：无

```cpp
RedisCommandBuilder builder;
builder.reserve(4, 6, 128);
builder.append("SET", std::array<std::string_view, 2>{"k1", "v1"});
builder.append("SET", std::array<std::string_view, 2>{"k2", "v2"});
builder.append("GET", std::array<std::string_view, 1>{"k1"});
builder.append("GET", std::array<std::string_view, 1>{"k2"});

auto batch_result = co_await client.batch(builder.commands()).timeout(std::chrono::seconds(10));
```

当前仓库里：

- `append(...)` 用来积累 batch
- `commands()` 返回 `std::span<const RedisCommandView>`
- `batch(...)` 才是对应的发送接口

## Pub/Sub

来源：`examples/include/E3-topology_pubsub_demo.cc`、`test/T11-topology_and_pubsub.cc`

- 目标：`E3-topology_pubsub_demo`
- 运行：`./<build-dir>/examples/E3-topology_pubsub_demo 127.0.0.1 6379`
- 环境变量：无

```cpp
auto subscribe_result = co_await subscriber.command(builder.subscribe(channel))
                            .timeout(std::chrono::seconds(5));
auto publish_result = co_await publisher.command(builder.publish(channel, message))
                          .timeout(std::chrono::seconds(5));
auto recv_result = co_await subscriber.receive(1).timeout(std::chrono::seconds(5));
```

收到的 Pub/Sub 消息是数组回复，通常检查：

- 下标 `0`：`"message"`
- 下标 `1`：channel
- 下标 `2`：payload

## timeout 与错误处理

来源：`test/T5-redis_client_timeout.cc`

- 目标：`T5-redis_client_timeout`
- 运行：`./<build-dir>/test/T5-redis_client_timeout`
- 环境变量：无

```cpp
auto result = co_await client.command(builder.get("test_key"))
                    .timeout(std::chrono::seconds(5));

if (!result) {
    std::cerr << result.error().message() << '\n';
}
```

建议统一分三层判断：

1. `!result`：I/O、协议或 timeout 错误
2. `result && !result.value()`：没有拿到完整回复
3. `result && result.value()`：解析 `RedisValue`

## 连接池

来源：`test/T4-connection_pool.cc`

- 目标：`T4-connection_pool`
- 运行：`./<build-dir>/test/T4-connection_pool`
- 环境变量：无

```cpp
auto pool_config = ConnectionPoolConfig::create("127.0.0.1", 6379, 2, 5);
pool_config.initial_connections = 2;

RedisConnectionPool pool(scheduler, pool_config);
auto init_result = co_await pool.initialize().timeout(std::chrono::seconds(5));
auto conn_result = co_await pool.acquire().timeout(std::chrono::seconds(5));
```

拿到连接后，底层仍然是 `RedisClient`：

```cpp
auto redis_client = conn_result.value()->get();
auto ping_result = co_await redis_client->command(builder.ping());
```

## TLS / `rediss://`

来源：`test/T17-rediss_client_tls.cc`、`test/T19-rediss_pool_and_topology.cc`

单节点 TLS 客户端：

```cpp
RedissClientConfig tls_config;
tls_config.ca_path = "/path/to/ca.pem";
tls_config.verify_peer = true;
tls_config.server_name = "redis.example.com";

auto client = RedissClientBuilder()
    .scheduler(scheduler)
    .tlsConfig(tls_config)
    .build();

auto connect_result = co_await client.connect("rediss://redis.example.com:6380/0").timeout(std::chrono::seconds(5));
auto ping_result = co_await client.command(RedisCommandBuilder().ping()).timeout(std::chrono::seconds(5));
```

TLS 连接池：

```cpp
auto pool_config = RedissConnectionPoolConfig::create("redis.example.com", 6380, 1, 4);
pool_config.tls_config = tls_config;

RedissConnectionPool pool(scheduler, pool_config);
auto init_result = co_await pool.initialize().timeout(std::chrono::seconds(5));
auto conn_result = co_await pool.acquire().timeout(std::chrono::seconds(5));
```

如果你只传 `rediss://host`，默认端口是 `6380`。SNI 默认使用 URL host；如果证书域名和接入地址不同，可以显式设置 `server_name`。

## 主从 / Sentinel

来源：`test/T11-topology_and_pubsub.cc`、`test/T13-integration_cluster_sentinel.cc`

- 本地 API 形状验证：`./<build-dir>/test/T11-topology_and_pubsub`
- 真实集成：`test/integration/run_cluster_sentinel_integration.sh --build-dir <build-dir>`

```cpp
auto ms = RedisMasterSlaveClientBuilder().scheduler(scheduler).build();
ms.setSentinelMasterName("mymaster");
ms.setAutoRetryAttempts(3);

RedisNodeAddress master;
master.host = "127.0.0.1";
master.port = 6379;

auto master_connect = co_await ms.connectMaster(master);
auto write_result = co_await ms.execute("SET", {"demo:ms:key", "ok"}, false, true);
auto read_result = co_await ms.execute("GET", {"demo:ms:key"}, true, true);
```

需要 Sentinel 时再额外添加：

```cpp
RedisNodeAddress sentinel;
sentinel.host = "127.0.0.1";
sentinel.port = 26379;

auto sentinel_connect = co_await ms.addSentinel(sentinel);
auto refresh_result = co_await ms.refreshFromSentinel();
```

TLS Sentinel 与 TCP 版本同构，只需要把 builder 换成 `RedissMasterSlaveClientBuilder` 并注入 `tlsConfig(...)`：

```cpp
auto ms = RedissMasterSlaveClientBuilder()
    .scheduler(scheduler)
    .tlsConfig(tls_config)
    .build();

auto sentinel_connect = co_await ms.addSentinel(sentinel).timeout(std::chrono::seconds(5));
auto refresh_result = co_await ms.refreshFromSentinel();
```

`refreshFromSentinel()` 现在返回 `Task<RedisCommandResult>`，直接 `co_await` 即可；不要再对这个方法本身调用 `.timeout(...)`。

## Cluster

来源：`test/T11-topology_and_pubsub.cc`、`test/T13-integration_cluster_sentinel.cc`

- API 形状验证：`./<build-dir>/test/T11-topology_and_pubsub`
- 真实 MOVED/ASK：`test/integration/run_cluster_sentinel_integration.sh --build-dir <build-dir>`

```cpp
auto cluster = RedisClusterClientBuilder().scheduler(scheduler).build();
cluster.setAutoRefreshInterval(std::chrono::milliseconds(1000));

RedisClusterNodeAddress seed;
seed.host = "127.0.0.1";
seed.port = 7000;
seed.slot_start = 0;
seed.slot_end = 16383;

auto add_seed = co_await cluster.addNode(seed);
auto refresh_slots = co_await cluster.refreshSlots();

const std::string key = "{user100}:name";
auto set_result = co_await cluster.execute("SET", {key, "alice"}, key, true);
auto get_result = co_await cluster.execute("GET", {key}, key, true);
```

TLS Cluster 入口同理：

```cpp
auto cluster = RedissClusterClientBuilder()
    .scheduler(scheduler)
    .tlsConfig(tls_config)
    .build();

auto add_seed = co_await cluster.addNode(seed).timeout(std::chrono::seconds(5));
auto refresh_slots = co_await cluster.refreshSlots();
```

对 Cluster 客户端来说，`routing_key` 是当前文档里的关键参数；旧文档里的 `executeByKey...` 名称已经不再使用。
