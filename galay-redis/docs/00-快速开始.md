# 00-快速开始

本页只覆盖当前仓库已经被示例和测试验证过的主路径：`RedisCommandBuilder` + `RedisClient::command(...)` / `RedisClient::batch(...)`。

## 先决条件

- CMake：`3.20+`
- 编译标准：`C++23`
- 外部依赖：`OpenSSL`、`spdlog`
- 内部依赖：`galay-kernel`、`galay-utils`
- 构建开关：`BUILD_EXAMPLES=ON`、`BUILD_TESTING=ON`（推荐）
- 兼容开关：`BUILD_TESTS` 仍可用，但只作为 `BUILD_TESTING` 的旧别名

```bash
cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=RelWithDebInfo \
  -DBUILD_EXAMPLES=ON \
  -DBUILD_TESTING=ON \
  -DBUILD_BENCHMARKS=OFF
cmake --build build --parallel
```

如果 `find_package(galay-kernel REQUIRED)` 或 `find_package(galay-utils REQUIRED)` 失败，请先安装这两个依赖，或通过 `CMAKE_PREFIX_PATH` 指向它们的 package config 目录。

如果你在当前 monorepo 联调，推荐优先使用共享前缀：

```bash
-DCMAKE_PREFIX_PATH=/Users/gongzhijie/Desktop/projects/git/.galay-prefix/latest
```

`galay-kernel` 当前对齐基线是 `3.4.4+`。

`examples/`、`test/`、`benchmark/` 现在不再写死 `/usr/local/include` 或 `/opt/homebrew`。如果依赖装在自定义前缀，请在配置阶段显式设置 `CMAKE_PREFIX_PATH`、`OpenSSL_ROOT_DIR` 或对应 package config 路径。

下文统一用 `<build-dir>` 表示你的 CMake binary dir；它可以是 `build`、`build-release` 等任意目录名。

## 第一个异步请求

来源：`examples/include/E1-async_basic_demo.cc`

- 目标：`E1-async_basic_demo`
- 运行：`./<build-dir>/examples/E1-async_basic_demo 127.0.0.1 6379`
- 环境变量：无

```cpp
#include "galay-redis/async/RedisClient.h"
#include <galay-kernel/kernel/Runtime.h>
#include <chrono>
#include <iostream>

using namespace galay::kernel;
using namespace galay::redis;

Coroutine run(IOScheduler* scheduler)
{
    auto client = RedisClientBuilder().scheduler(scheduler).build();
    RedisCommandBuilder builder;

    auto connect_result = co_await client.connect("127.0.0.1", 6379).timeout(std::chrono::seconds(5));
    if (!connect_result) {
        std::cerr << connect_result.error().message() << '\n';
        co_return;
    }

    auto set_result = co_await client.command(builder.set("quick:key", "hello")).timeout(std::chrono::seconds(5));
    if (!set_result || !set_result.value()) {
        std::cerr << "SET failed\n";
        (void)co_await client.close();
        co_return;
    }

    auto get_result = co_await client.command(builder.get("quick:key")).timeout(std::chrono::seconds(5));
    if (get_result && get_result.value()) {
        const auto& values = get_result.value().value();
        if (!values.empty() && values[0].isString()) {
            std::cout << values[0].toString() << '\n';
        }
    }

    (void)co_await client.close();
}
```

如果模块工具链满足条件，`examples/import/E1-async_basic_demo.cc` 还提供了同场景的 `E1-async_basic_demo-import`，用于验证 `import galay.redis;` 的消费路径。

## 返回值速记

### `RedisClient::command(...)` / `RedisClient::batch(...)`

`co_await` 之后的结果类型是：

```cpp
std::expected<std::optional<std::vector<RedisValue>>, RedisError>
```

- `!result`：命令失败，查看 `result.error()`
- `result && !result.value()`：awaitable 没有产出完整回复，示例代码会把它视为异常/空响应
- `result && result.value()`：得到 `std::vector<RedisValue>`

### `RedisMasterSlaveClient::execute(...)` / `RedisClusterClient::execute(...)`

`co_await` 之后的结果类型是：

```cpp
std::expected<std::vector<RedisValue>, RedisError>
```

拓扑客户端不会再包一层 `std::optional`。

## 批量发送最短写法

来源：`examples/include/E2-pipeline_demo.cc`、`test/T8-redis_batch_timeout_api.cc`、`test/T9-redis_batch_span_api.cc`

- 目标：`E2-pipeline_demo`
- 运行：`./<build-dir>/examples/E2-pipeline_demo 127.0.0.1 6379 demo:pipeline: 20`
- 环境变量：无

```cpp
RedisCommandBuilder builder;
builder.reserve(2, 3, 64);
builder.append("SET", std::array<std::string_view, 2>{"batch:key", "v1"});
builder.append("GET", std::array<std::string_view, 1>{"batch:key"});

auto result = co_await client.batch(builder.commands()).timeout(std::chrono::seconds(5));
```

当前 async 文档里不再使用旧的 `client.pipeline(...)` 说法；源码中真实存在的是 `batch(...)`。

## 接入已有 CMake 工程

当前仓库已经导出可安装的 CMake package config，企业项目推荐：

```cmake
find_package(galay-redis CONFIG REQUIRED)
target_link_libraries(your_app PRIVATE galay-redis::galay-redis)
```

如果你在 monorepo 或源码树中直接联调，也可以继续 vendoring：

```cmake
add_subdirectory(external/galay-redis)
target_link_libraries(your_app PRIVATE galay-redis)
```

## 当前不应再沿用的旧写法

以下写法在当前 async 公开头文件中都不是主接口：

- `co_await client.get("key")`
- `co_await client.set("key", "value")`
- `co_await client.pipeline(commands)`
- `cluster.executeByKeyAuto(...)`

请统一迁移到：

- `client.command(builder.get("key"))`
- `client.command(builder.set("key", "value"))`
- `client.batch(builder.commands())`
- `cluster.execute("GET", {key}, routing_key, auto_retry)`
