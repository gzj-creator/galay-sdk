# 00-快速开始

本页只覆盖“把项目编出来并跑通第一个真实示例”所需的最小信息。

## 1. 前置条件

| 类别 | 要求 | 备注 |
| --- | --- | --- |
| 编译器 | 支持 C++23 | 由 `GALAY_ETCD_CXX_STANDARD` 控制，默认 `23` |
| CMake | `>= 3.20` | 顶层 `CMakeLists.txt` 要求 |
| 内部依赖 | `galay-kernel`、`galay-utils`、`GalayHttp` | 均为 `find_package(... REQUIRED)` |
| 第三方依赖 | `spdlog`、`simdjson` | `simdjson` 通过 `pkg-config` 查找 |
| 本地服务 | etcd v3 HTTP endpoint | 推荐显式传 `http://127.0.0.1:2379` |

建议先确认 etcd endpoint 可用：

```bash
curl http://127.0.0.1:2379/version
```

## 2. 构建库、示例、测试与 benchmark

```bash
cmake -S . -B build \
  -D GALAY_ETCD_BUILD_EXAMPLES=ON \
  -D BUILD_TESTING=ON \
  -D GALAY_ETCD_BUILD_BENCHMARKS=ON

cmake --build build -j
```

相关开关：

- `GALAY_ETCD_BUILD_EXAMPLES`：生成 `E1-SyncBasic`、`E2-AsyncBasic`（及可选 import 版本）
- `BUILD_TESTING`：生成测试目标（`T1`~`T8`）并注册到 `ctest`
- `BUILD_TESTS` / `GALAY_ETCD_BUILD_TESTS`：`BUILD_TESTING` 兼容别名
- `GALAY_ETCD_BUILD_BENCHMARKS`：生成 `B1-EtcdKvBenchmark`
- `GALAY_ETCD_ENABLE_IMPORT_COMPILATION`：尝试启用 C++ module/import 编译路径

## 3. 先跑真实示例

真实文件与 target：

| 文件 | target | 命令 | 环境变量 |
| --- | --- | --- | --- |
| `examples/include/E1-sync_basic.cc` | `E1-SyncBasic` | `./build/examples/E1-SyncBasic http://127.0.0.1:2379` | 无 |
| `examples/include/E2-async_basic.cc` | `E2-AsyncBasic` | `./build/examples/E2-AsyncBasic http://127.0.0.1:2379` | 无 |
| `examples/import/E1-sync_basic.cc` | `E1-SyncBasicImport` | `./build/examples/E1-SyncBasicImport http://127.0.0.1:2379` | 无 |
| `examples/import/E2-async_basic.cc` | `E2-AsyncBasicImport` | `./build/examples/E2-AsyncBasicImport http://127.0.0.1:2379` | 无 |

说明：

- 两个示例都优先从 `argv[1]` 读取 endpoint；未传时回落到 `http://127.0.0.1:2379`
- 企业环境 / CI 仍建议始终显式传入自己的 endpoint

## 4. 在你的工程里链接

### 4.1 作为子目录

```cmake
add_subdirectory(path/to/galay-etcd)
target_link_libraries(your_app PRIVATE galay-etcd)
```

### 4.2 作为安装包

安装后使用：

```cmake
find_package(galay-etcd CONFIG REQUIRED)
target_link_libraries(your_app PRIVATE galay-etcd::galay-etcd)
```

使用 module facade target 时：

```cmake
find_package(galay-etcd CONFIG REQUIRED)
target_link_libraries(your_app PRIVATE galay-etcd::galay-etcd-modules)
```

安装包会继续要求找到这些依赖：

- `galay-kernel`
- `galay-utils`
- `GalayHttp`
- `spdlog`

## 5. 最小同步路径

下面的片段与 `examples/include/E1-sync_basic.cc` 保持一致的 API 形态：

```cpp
#include "galay-etcd/sync/EtcdClient.h"
#include <iostream>

int main() {
    galay::etcd::EtcdConfig config;
    config.endpoint = "http://127.0.0.1:2379";

    auto client = galay::etcd::EtcdClientBuilder().config(config).build();
    if (auto conn = client.connect(); !conn) {
        std::cerr << conn.error().message() << '\n';
        return 1;
    }

    if (auto put = client.put("/quickstart/key", "value"); !put) {
        std::cerr << put.error().message() << '\n';
        return 1;
    }

    if (auto get = client.get("/quickstart/key"); !get) {
        std::cerr << get.error().message() << '\n';
        return 1;
    }

    (void)client.del("/quickstart/key");
    (void)client.close();
    return 0;
}
```

## 6. 最小异步路径

异步客户端必须绑定 `IOScheduler*`；完整可运行版本请直接看 `examples/include/E2-async_basic.cc`。

```cpp
Coroutine run(galay::kernel::IOScheduler* scheduler) {
    galay::etcd::AsyncEtcdConfig config;
    config.endpoint = "http://127.0.0.1:2379";

    auto client = galay::etcd::AsyncEtcdClientBuilder()
        .scheduler(scheduler)
        .config(config)
        .build();

    if (auto conn = co_await client.connect(); !conn) {
        co_return;
    }

    (void)co_await client.put("/quickstart/async", "value");
    (void)co_await client.close();
}
```

## 7. module / import 路径

公开 module 文件是 `galay-etcd/module/galay.etcd.cppm`，导出名为：

```cpp
import galay.etcd;
```

但“安装 `.cppm` 文件”与“成功启用 import 编译”不是一回事。当前仓库只有在以下条件同时满足时才会真正启用 module/import 编译路径：

- `CMake >= 3.28`
- 生成器为 `Ninja` 或 `Visual Studio`
- `Linux + GCC >= 14`，或非 `AppleClang` 且存在 `clang-scan-deps`

不满足条件时，头文件路径仍然是权威入口。
