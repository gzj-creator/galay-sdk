# 04-示例代码

本页只列真实存在的文件、真实存在的 target 和真实可执行命令。

## 1. 运行约定

- 环境变量：当前公开示例 / 测试 / benchmark 都不要求环境变量
- endpoint：除 `T6-EtcdInternalHelpers` 外，统一支持通过 `argv[1]` 传入；未传时回落到 `http://127.0.0.1:2379`
- 推荐命令：始终显式传 `http://127.0.0.1:2379` 或你的企业 endpoint

| 状态 | 源文件 | target | 覆盖内容 | 运行命令 |

- include/import 文件都是真实样例，不是伪代码

- import 示例只有在 `GALAY_ETCD_IMPORT_COMPILATION_ENABLED=ON` 且 `galay-etcd-modules` 可用时才会生成

## 5. 说明性模式（非 examples/、非 tests/）

以下内容只存在于文档中，不对应 `examples/` 或 `test/` target，因此必须视为“说明性模式”，不是“已验证示例”：

- 服务注册 / 服务发现
- 配置中心
- 简化版分布式锁

这些模式集中放在 [06-高级主题](06-高级主题.md)。如果你要把它们迁入生产代码，建议先：

1. 为模式补充独立 `examples/` 或 `test/`
2. 明确 TLS / 认证 / 条件事务等前置能力是否已经实现
3. 给出真实运行命令和验收标准

## 6. 选择建议

- 想看最短可运行代码：先跑 `E1-SyncBasic` / `E2-AsyncBasic`
- 想验证 prefix：跑 `T2-EtcdPrefixOps`
- 想验证 pipeline：跑 `T3-EtcdPipeline` / `T5-AsyncEtcdPipeline`
- 想做无外部依赖的本地校验：跑 `T6-EtcdInternalHelpers`
