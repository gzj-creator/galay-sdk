# 文档总览

本文档索引给出当前仓库的规范阅读顺序，并把每页与真实源码范围对应起来。

## 推荐阅读顺序

1. [00-快速开始](00-快速开始.md) - 依赖、构建、首次验证命令
2. [01-架构设计](01-架构设计.md) - 结构与职责划分
3. [02-API参考](02-API参考.md) - 公开头文件、核心类型、API 签名、失败语义与示例 / 测试锚点
4. [03-使用指南](03-使用指南.md) - 使用模式与实践建议
5. [04-示例代码](04-示例代码.md) - 映射到真实 `examples/` / `test/` 的示例索引
6. [05-性能测试](05-性能测试.md) - benchmark 目标、命令与当前证据状态
7. [06-高级主题](06-高级主题.md) - 模块构建、运行时模型与扩展边界
8. [07-常见问题](07-常见问题.md) - 当前限制与排障

## 内容约束

- 文件编号、链接编号、页面标题编号保持一致。
- 示例优先映射到真实文件；映射不到的内容必须明确标成“示意 / 未验证”。
- Benchmark 页面只在存在真实命令、参数与结果来源时给出数字；否则仅保留验证方式。
- 文档与源码冲突时，优先以公开头文件、实现、示例、测试、benchmark 为准。
- 测试 / 回归说明统一收敛到 `00-快速开始`、`04-示例代码` 与 `07-常见问题`，避免拆出重复测试页影响 RAG 命中。

## 快速定位

- 想先跑通一次：看 [00-快速开始](00-快速开始.md)
- 想查 API、前置条件或失败语义：看 [02-API参考](02-API参考.md)
- 想找示例文件 / target / 命令：看 [04-示例代码](04-示例代码.md)
- 想确认性能页到底是不是当前结果：看 [05-性能测试](05-性能测试.md)
- 想跑回归测试：看 [04-示例代码](04-示例代码.md) 的测试 / 脚本入口与 [07-常见问题](07-常见问题.md) 的排障说明
