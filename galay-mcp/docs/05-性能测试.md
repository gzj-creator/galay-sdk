## 7. Rust HTTP 基线与 `S6`

为了满足“同机 / 同构建 / 同 workload”的基线硬要求，新增 `scripts/S6-RunRustCompare.sh`：

1. `S6` 依赖本仓库已有的 `build/`（C++ target B2/B3）和 Rust toolchain（`cargo`）。脚本会进入 `benchmark/compare/rust/`、执行 `cargo build --locked --release`，启动 `[package] name = "galay-rust-benchmark"` 的 Axum/Hyper/Tokio HTTP 服务（默认监听 `127.0.0.1:8081`）。
2. 启动后会轮询 `http://127.0.0.1:8081/mcp` 确认就绪，把日志写入 `/tmp/galay-rust-baseline.log`，并打印 `uname -a`、`rustc --version`、`cargo --version` 作为环境快照。
3. `S6` 默认优先复用可写的 `~/.cargo`；若默认 `~/.cargo` 不可写，会自动回退到 `${TMPDIR:-/tmp}/galay-mcp-cargo-home`，并在可用时复制 `~/.cargo/config.toml` 作为镜像配置。也可以显式传入 `CARGO_HOME`、`CARGO_TARGET_DIR`、`RUSTC_BIN` 覆盖默认行为。
4. Server 建立后，脚本会依次调用 `build/bin/B2-http_performance` 和 `build/bin/B3-concurrent_requests`，将 `--url` 指向 Rust 服务，并使用当前公开 workload 默认值：`B2 connections=8 requests=2000 io=2 compute=0`，`B3 workers=10 requests=100`。脚本退出时会自动清理服务器。
5. 默认可以通过 `RUST_PORT` 来调整端口：

```bash
RUST_PORT=9080 scripts/S6-RunRustCompare.sh
```

6. 任何对外发布的 HTTP benchmark（B2/B3）必须包含该脚本产生的 Rust baseline 命令、stdout/stderr、`/tmp/galay-rust-baseline.log` 以及对应的 C++ 输出；没有 Rust baseline 的数字仍属 internal-only / historical。

`benchmark/compare/rust/src/main.rs` 是一个最小的 Axum 服务器：它解析 JSON-RPC 请求，记录 `method` 与 `id`，并回 `{"jsonrpc":"2.0","id":...,"result":{"status":"ok","method":...}}`。`scripts/S6` 只是让 B2/B3 运行在这个服务上，保持与 C++ HTTP MCP 相同的 workload。

由于没有公开的 Rust stdio MCP 版本，`B1-stdio_performance` 继续只用作内部历史记录；任何 B1 数字都必须标注为 internal-only / historical，并明确缺乏 Rust stdio 对照。
