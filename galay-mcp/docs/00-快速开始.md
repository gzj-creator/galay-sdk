# 00-快速开始

本页只保留可直接对应到当前仓库的依赖、构建命令和首次验证路径。

## 1. 版本与依赖

### 编译要求

- CMake：`>= 3.20`
- C++：`C++23`
- 模块示例：`CMake >= 3.28`，且生成器需支持模块扫描（`Ninja` / `Visual Studio`）

### 当前构建图的真实依赖

| 构建场景 | 必需外部依赖 | 说明 |
| --- | --- | --- |
| 默认 `galay-mcp` 库 | `simdjson`、`galay-kernel`、`galay-http` | 当前库 target 会统一编译 `client/*.cc` 与 `server/*.cc`，HTTP 头文件无条件出现在公开 API 中 |
| `stdio` 示例 / 测试 | 同上 | 当前没有单独的 `stdio-only` 库开关 |
| HTTP 示例 / 测试 / benchmark | 同上 | 需要 HTTP / Runtime 相关依赖 |
| 模块示例 | 同上 + 模块工具链 | 会额外生成 `galay-mcp-modules` |

### 安装示例

```bash
# macOS (Homebrew)
brew install cmake simdjson

# Ubuntu / Debian
sudo apt-get update
sudo apt-get install -y cmake g++ libsimdjson-dev
```

`galay-kernel` 与 `galay-http` 需要按其各自仓库完成安装，并保证头文件和库对 CMake 可见。

## 2. 配置与构建

```bash
cmake -S . -B build -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build
```

常见变体：

```bash
# 关闭 examples / tests / benchmarks
cmake -S . -B build \
  -DBUILD_EXAMPLES=OFF \
  -DBUILD_TESTING=OFF \
  -DBUILD_BENCHMARKS=OFF

# 尝试启用模块示例
cmake -S . -B build -DBUILD_MODULE_EXAMPLES=ON -G Ninja
```

`BUILD_TESTS` 仍可用于兼容旧脚本，但标准 CTest / CI 开关是 `BUILD_TESTING`。

## 3. 第一次验证：stdio 服务端协议

这个命令只依赖 `T2-stdio_server`，适合确认最小协议链路：

```bash
printf '%s\n' \
  '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{"protocolVersion":"2024-11-05","clientInfo":{"name":"quickstart","version":"1.0.0"},"capabilities":{}}}' \
  | ./build/bin/T2-stdio_server
```

预期输出是单行 JSON-RPC 响应，包含 `"result"`、`"serverInfo"` 和 `"capabilities"` 字段。

## 4. 第一次双向联调：stdio 客户端 + 服务端

`stdio` 联调需要双向管道。**不要使用单个 `|`**。

```bash
mkfifo /tmp/galay-mcp-c2s /tmp/galay-mcp-s2c
./build/bin/T2-stdio_server < /tmp/galay-mcp-c2s > /tmp/galay-mcp-s2c &
SERVER_PID=$!
./build/bin/T1-stdio_client > /tmp/galay-mcp-c2s < /tmp/galay-mcp-s2c
kill ${SERVER_PID}
rm -f /tmp/galay-mcp-c2s /tmp/galay-mcp-s2c
```

仓库脚本：`scripts/S4-RunIntegrationTest.sh`

## 5. HTTP 联调

终端 1：

```bash
./build/bin/T4-http_server 8080 0.0.0.0
```

终端 2：

```bash
./build/bin/T3-http_client http://127.0.0.1:8080/mcp
```

默认 URL 与 `test/T3-http_client.cc` 一致：`http://127.0.0.1:8080/mcp`。

## 6. 运行 examples

### Stdio 示例

- 来源：`examples/include/E1-basic_stdio_usage.cc`
- target：`E1-BasicStdioUsage`
- 必需环境变量：无

```bash
./build/bin/E1-BasicStdioUsage server
./build/bin/E1-BasicStdioUsage client
```

如果要让客户端与服务端真正对接，也需要像上面一样使用双向 FIFO。

### HTTP 示例

- 来源：`examples/include/E2-basic_http_usage.cc`
- target：`E2-BasicHttpUsage`
- 必需环境变量：无

终端 1：

```bash
./build/bin/E2-BasicHttpUsage server
```

终端 2：

```bash
./build/bin/E2-BasicHttpUsage client http://127.0.0.1:8080/mcp
```

## 7. 模块示例

满足模块工具链条件时，会生成以下 target：

- `E1-BasicStdioUsageImport`
- `E2-BasicHttpUsageImport`
- `galay-mcp-modules`

模块名为：

```cpp
import galay.mcp;
```

如果 CMake 检测到模块扫描不受支持，`BUILD_MODULE_EXAMPLES` 会被自动关闭，并给出原因。
