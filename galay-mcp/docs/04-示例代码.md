# 04-示例代码

本页只收录能直接映射到当前仓库真实文件的示例。所有示例都给出：来源文件、target、运行命令、环境变量和验证状态。

## 收录规则

文档中的公开示例必须满足以下至少一项：

- 直接映射到 `examples/` 中的真实文件
- 直接映射到 `test/` 中的真实文件
- 被明确标记为“示意 / 未验证”

本页已经移除了此前混入的占位函数、未实现 transport 和伪业务代码。

## 1. `examples/` 中的真实示例

### 1.1 基础 Stdio 示例（include 版本）

- 来源文件：`examples/include/E1-basic_stdio_usage.cc`
- 共享实现：`examples/common/E1-BasicStdioUsageMain.inc`
- CMake target：`E1-BasicStdioUsage`
- 必需环境变量：无
- 依赖：`simdjson`、`galay-kernel`、`galay-http`（因为当前库构建图统一编译 HTTP / stdio 源文件）

构建：

```bash
cmake -S . -B build -DBUILD_MODULE_EXAMPLES=OFF
cmake --build build --target E1-BasicStdioUsage
```

运行模式：

```bash
./build/bin/E1-BasicStdioUsage server
./build/bin/E1-BasicStdioUsage client
```

如需真实双向联调，请使用双向 FIFO：

```bash
mkfifo /tmp/galay-e1-c2s /tmp/galay-e1-s2c
./build/bin/E1-BasicStdioUsage server < /tmp/galay-e1-c2s > /tmp/galay-e1-s2c &
SERVER_PID=$!
./build/bin/E1-BasicStdioUsage client > /tmp/galay-e1-c2s < /tmp/galay-e1-s2c
kill ${SERVER_PID}
rm -f /tmp/galay-e1-c2s /tmp/galay-e1-s2c
```

示例覆盖：

- `McpStdioServer::setServerInfo`
- `McpStdioServer::addTool`
- `McpStdioServer::addResource`
- `McpStdioClient::initialize`
- `McpStdioClient::listTools`
- `McpStdioClient::callTool`
- `McpStdioClient::listResources`
- `McpStdioClient::readResource`

## 2. `test/` 中可直接复用的示例

这些测试程序本身就是最可靠的可运行样例，因为它们直接对齐当前 target 名称和公开 API。

| 来源文件 | target | 场景 | 运行命令 | 必需环境变量 |
| --- | --- | --- | --- | --- |
| `test/T1-stdio_client.cc` | `T1-stdio_client` | stdio 客户端 | `./build/bin/T1-stdio_client` | 无 |
| `test/T2-stdio_server.cc` | `T2-stdio_server` | stdio 服务端 | `./build/bin/T2-stdio_server` | 无 |
| `test/T3-http_client.cc` | `T3-http_client` | HTTP 客户端 | `./build/bin/T3-http_client http://127.0.0.1:8080/mcp` | 无 |
| `test/T4-http_server.cc` | `T4-http_server` | HTTP 服务端 | `./build/bin/T4-http_server 8080 0.0.0.0` | 无 |

### 2.1 Stdio 测试对跑

```bash
mkfifo /tmp/galay-test-c2s /tmp/galay-test-s2c
./build/bin/T2-stdio_server < /tmp/galay-test-c2s > /tmp/galay-test-s2c &
SERVER_PID=$!
./build/bin/T1-stdio_client > /tmp/galay-test-c2s < /tmp/galay-test-s2c
kill ${SERVER_PID}
rm -f /tmp/galay-test-c2s /tmp/galay-test-s2c
```

### 2.2 HTTP 测试对跑

终端 1：

```bash
./build/bin/T4-http_server 8080 0.0.0.0
```

终端 2：

```bash
./build/bin/T3-http_client http://127.0.0.1:8080/mcp
```

### 2.3 Stdio 脚本化回归入口

#### `scripts/S2-Run.sh`：协议级服务端校验

- 真相来源：`test/T2-stdio_server.cc`
- 依赖 target：`T2-stdio_server`
- 运行命令：`BUILD_DIR=build bash scripts/S2-Run.sh`
- 必需环境变量：`BUILD_DIR`（可选，默认 `build`）

当前脚本会顺序验证：

- `initialize`
- `tools/list`
- `tools/call`（`add`）
- `resources/list`
- `resources/read`（`file:///test.txt`）
- `prompts/list`
- `prompts/get`（`write_essay`）
- `ping`

脚本断言的真实注册对象来自 `test/T2-stdio_server.cc`：

- 工具：`add`、`concat`
- 资源：`file:///test.txt`
- 提示：`write_essay`

#### `scripts/S4-RunIntegrationTest.sh`：stdio 双向联调

- 真相来源：`test/T1-stdio_client.cc`、`test/T2-stdio_server.cc`
- 依赖 target：`T1-stdio_client`、`T2-stdio_server`
- 运行命令：`BUILD_DIR=build bash scripts/S4-RunIntegrationTest.sh`
- 必需环境变量：`BUILD_DIR`（可选，默认 `build`）

该脚本使用两条 FIFO 建立双向 `stdio` 会话，不依赖单个 pipe。它是 `T1-stdio_client` 与 `T2-stdio_server` 的首选联调入口。

### 2.4 HTTP 脚本化回归入口

#### `scripts/S5-TestHttpServer.sh`：服务端 JSON-RPC 校验

- 真相来源：`test/T4-http_server.cc`
- 依赖 target：`T4-http_server`
- 运行命令：`HOST=127.0.0.1 PORT=8080 BUILD_DIR=build bash scripts/S5-TestHttpServer.sh`
- 必需环境变量：无

脚本覆盖：

- `initialize`
- `tools/list`
- `tools/call`（`echo`、`add`）
- `resources/list`
- `resources/read`（`example://hello`）
- `prompts/list`
- `prompts/get`（`greeting`）
- `ping`

脚本断言的真实注册对象来自 `test/T4-http_server.cc`：

- 工具：`echo`、`add`
- 资源：`example://hello`、`example://info`
- 提示：`greeting`

#### `scripts/S7-RunHttpIntegrationTest.sh`：HTTP 客户端 / 服务端联调

- 真相来源：`test/T3-http_client.cc`、`test/T4-http_server.cc`
- 依赖 target：`T3-http_client`、`T4-http_server`
- 运行命令：`BUILD_DIR=build bash scripts/S7-RunHttpIntegrationTest.sh`
- 必需环境变量：`BUILD_DIR`、`HOST`、`PORT`（均可选）

脚本会先启动 `T4-http_server`，再通过 `wait_for_server` 轮询可达性，然后运行 `T3-http_client`。`test/T3-http_client.cc` 当前覆盖：

- `connect`
- `initialize`
- `ping`
- `listTools`
- `callTool`（`echo`、`add`）
- `listResources`
- `readResource`
- `listPrompts`
- `getPrompt`
- `disconnect`

## 3. 不再作为“已验证示例”收录的内容

以下内容不应再被当成可直接运行的示例：

- 依赖未提供实现的占位业务函数
- 夹杂“添加结果行...”等说明性占位文本的伪代码
- WebSocket transport 草图
- 未映射到 `examples/` / `test/` / 公开头文件的接口片段

如果未来确实要增加概念性草图，请显式标注为“示意 / 未验证”，并与真实示例分栏展示。
