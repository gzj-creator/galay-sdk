## 构建前置条件

- CMake `>= 3.16`
- C++23 编译器
- 已正确安装/可发现 `galay-kernel`
- 配置时启用 `BUILD_BENCHMARKS=ON`
- 推荐使用 `Release` 构建

推荐构建命令：

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --target \
  B1-RpcBenchServer \
  B2-RpcBenchClient \
  B3-ServiceDiscoveryBench \
  B4-RpcStreamBenchServer \
  B5-RpcStreamBenchClient
```

## Benchmark 清单

### `B1-RpcBenchServer`

- 源文件：`benchmark/B1-rpc_bench_server.cpp`
- 作用：请求/响应 Echo 压测服务端
- 运行命令：`./build/benchmark/B1-RpcBenchServer [port] [io_scheduler_count] [ring_buffer_size]`
- 默认值：`9000` / `0`（自动） / `131072`
- 必需环境变量：无
- 说明：当 `io_scheduler_count=0` 时，程序内部会映射到 `GALAY_RUNTIME_SCHEDULER_COUNT_AUTO`

### `B2-RpcBenchClient`

- 源文件：`benchmark/B2-rpc_bench_client.cpp`
- 作用：请求/响应压测客户端，统计请求数、吞吐与延迟分位数
- 运行命令：`./build/benchmark/B2-RpcBenchClient [options]`
- 关键参数：
  - `-h <host>`：服务器地址，默认 `127.0.0.1`
  - `-p <port>`：服务器端口，默认 `9000`
  - `-c <connections>`：连接数，默认 `100`
  - `-s <size>`：payload 字节数，默认 `256`
  - `-d <duration>`：测试秒数，默认 `10`
  - `-i <io_count>`：IO 调度器数量，默认 `0`（自动）
  - `-l <pipeline>`：每连接 pipeline 深度，默认 `1`
  - `-m <mode>`：`unary | client_stream | server_stream | bidi`
- 必需环境变量：无
- 自动验证：`RpcBenchmarkSmokeTest`（轻载参数）

### `B3-ServiceDiscoveryBench`

- 源文件：`benchmark/B3-service_discovery_bench.cpp`
- 作用：`AsyncLocalServiceRegistry` 的本地内存注册/发现/注销基准
- 运行命令：`./build/benchmark/B3-ServiceDiscoveryBench -w 100 -d 10 -i 2`
- 关键参数：
  - `-w`：worker 数
  - `-d`：测试持续时间（秒）
  - `-i`：IO 调度器数量
- 必需环境变量：无
- 注意：这是本地注册中心 benchmark，不代表 etcd / Consul 等外部注册中心性能
- Rust baseline：当前尚无公平的 Rust 对照实现，所以 `B3-ServiceDiscoveryBench` 的输出只能标记为 internal-only / historical，不能作为对外发布的基准数据。

### `B4-RpcStreamBenchServer`

- 源文件：`benchmark/B4-rpc_stream_bench_server.cpp`
- 作用：真实 `STREAM_*` 协议压测服务端
- 运行命令：`./build/benchmark/B4-RpcStreamBenchServer [port] [io_scheduler_count] [ring_buffer_size]`
- 默认值：`9100` / `0`（自动） / `131072`
- 必需环境变量：无
- 说明：当 `io_scheduler_count=0` 时，程序内部会映射到 `GALAY_RUNTIME_SCHEDULER_COUNT_AUTO`

### `B5-RpcStreamBenchClient`

- 源文件：`benchmark/B5-rpc_stream_bench_client.cpp`
- 作用：真实流 benchmark 客户端，统计 Streams/s、Frames/s、吞吐与延迟
- 运行命令：`./build/benchmark/B5-RpcStreamBenchClient [options]`
- 关键参数：
  - `-h <host>`：服务器地址，默认 `127.0.0.1`
  - `-p <port>`：服务器端口，默认 `9100`
  - `-c <connections>`：连接数，默认 `100`
  - `-s <size>`：每帧 payload 字节数，默认 `128`
  - `-f <frames>`：每条 stream 的帧数，默认 `16`
  - `-w <window>`：每条 stream 的帧级 pipeline 窗口，默认 `1`
  - `-d <duration>`：测试秒数，默认 `10`
  - `-i <io_count>`：IO 调度器数量，默认 `0`（自动）
- 必需环境变量：无
- 自动验证：`RpcStreamBenchmarkSmokeTest`（轻载参数）

## 可复现实验命令

### 请求/响应基准

```bash
# 终端 1
./build/benchmark/B1-RpcBenchServer 9000 0 131072

# 终端 2：47B 小包，四种模式对照
./build/benchmark/B2-RpcBenchClient -h 127.0.0.1 -p 9000 -c 200 -d 5 -s 47 -i 0 -l 4 -m unary
./build/benchmark/B2-RpcBenchClient -h 127.0.0.1 -p 9000 -c 200 -d 5 -s 47 -i 0 -l 4 -m client_stream
./build/benchmark/B2-RpcBenchClient -h 127.0.0.1 -p 9000 -c 200 -d 5 -s 47 -i 0 -l 4 -m server_stream
./build/benchmark/B2-RpcBenchClient -h 127.0.0.1 -p 9000 -c 200 -d 5 -s 47 -i 0 -l 4 -m bidi

# 64KB 大包
./build/benchmark/B2-RpcBenchClient -h 127.0.0.1 -p 9000 -c 100 -d 5 -s 65536 -i 0 -l 1 -m unary
```

### 本地服务发现基准

```bash
./build/benchmark/B3-ServiceDiscoveryBench -w 100 -d 10 -i 2
```

### 真实流基准

```bash
# 终端 1
./build/benchmark/B4-RpcStreamBenchServer 9100 0 131072

# 终端 2
./build/benchmark/B5-RpcStreamBenchClient -h 127.0.0.1 -p 9100 -c 100 -d 5 -s 128 -f 16 -w 8 -i 0
```

## Rust 对标与对外发布边界

- 使用 `scripts/S3-Bench-Rust-Compare.sh` 可在本机按同参数对照 `B1/B2`（`unary | client_stream | server_stream | bidi`）与 `B4/B5`（stream benchmark 映射到 tonic `stream_bench` 模式）。
- `B3-ServiceDiscoveryBench` 目前仍无公平的主流 Rust 对照实现，因此只能归类为 `internal-only` / `historical`，不能出现在对外对比表格或宣传材料中。

没有这些元数据时，数字只能视为历史观察值，不能作为当前版本的企业级可审计结论。
