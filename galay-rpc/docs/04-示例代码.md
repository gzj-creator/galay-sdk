# 04-示例代码

本页只收录能够直接映射到仓库真实文件、真实 target 和真实运行命令的示例。说明性片段已移出本页，避免把“思路代码”误当成仓库自带示例。

## 前置条件

- CMake `>= 3.16`
- C++23 编译器
- 已满足 `README.md` / `docs/00-快速开始.md` 中的 `galay-kernel` 依赖要求
- 本页所有运行命令均默认从仓库根目录执行
- `examples/` 需要 `BUILD_EXAMPLES=ON`
- `test/` 需要 `BUILD_TESTING=ON`
- import 版本还需要：
  - `BUILD_MODULE_EXAMPLES=ON`
  - CMake `>= 3.28`
  - 支持 C++ 模块的生成器（`Ninja` / `Visual Studio`）

## `examples/include/`：可直接构建运行的 include 版本

### `E1-EchoServer`

- 源文件：`examples/include/E1-echo_server.cc`
- CMake target：`E1-EchoServer`
- 构建开关：`BUILD_EXAMPLES=ON`
- 运行命令：`./build/examples/E1-EchoServer 9000`
- 必需环境变量：无
- 已验证事实：注册 `EchoService`，其中 `echo` 同时覆盖 `unary` / `client_stream` / `server_stream` / `bidi` 四种调用模式
- 自动验证：`EchoExampleSmokeTest`

### `E2-EchoClient`

- 源文件：`examples/include/E2-echo_client.cc`
- CMake target：`E2-EchoClient`
- 构建开关：`BUILD_EXAMPLES=ON`

- 必需环境变量：无
- 已验证事实：演示 `call(...)`、`callClientStreamFrame(...)`、`callServerStreamRequest(...)`、`callBidiStreamFrame(...)`

## 如何区分“仓库示例”和“说明性片段”

- 本页中的每一项都必须能命中真实文件和真实 target。
- 需要了解外部注册中心、熔断、限流、监控或 HTTP/JSON 集成时，请查看 `docs/06-高级主题.md`。
- `docs/06-高级主题.md` 中未映射到 `examples/` / `test/` / `benchmark/` 的片段，均应按“说明性集成思路”理解，而不是仓库自带实现。
