## 当前验收口径

- core benchmark 允许 `5%` 以内回退，视为调度噪声
- non-core benchmark 维持 `10%` 容忍
- 最终结论只接受 fresh `baseline -> refactored` 串行重跑数据，不接受历史三态结果复用

## 当前执行约束

- `B5-UdpClient` 只做 smoke / stability 检查
- 最终 UDP 性能签收使用 `B6-Udp`
- baseline 的 `io_uring` `B4/B5` 历史 UDP 路径显式跳过
- baseline 的 `io_uring` `B6-Udp` 必须显式使用真实 `io_uring` 构建，不允许静默降级到 `epoll`

## 当前已落地并已完成 fresh 回归确认的补充点

1. `B8` latency benchmark 增加 consumer-ready barrier
2. `MpscChannel` 增加 recv-side single-consumer prefetch
3. `MpscChannel` 的默认 batch size / prefetch limit 改为构造参数配置
4. `B6-Udp` 增加尾部 drain，降低 packet loss 抖动
5. `B1` latency 改为使用 `min(scheduler_count, 2)`，避免 burst 队列长度淹没调度开销
6. `B4-UdpServer` 将统计逻辑移出 scheduler 协程，避免阻塞事件循环并恢复 `B4/B5` 有效收发

## 推荐复现命令

单 benchmark 全后端串行：

```bash
bash scripts/run_single_benchmark_triplet.sh \
  --baseline-ref cde3da1 \
  --refactored-path "$PWD" \
  --benchmark B6-Udp \
  --repeat 3 \
  --output-root "$PWD/build/single-benchmark-triplet-2026-03-15-final/B6-Udp"
```

单 backend 低层执行：

```bash
bash scripts/run_benchmark_triplet.sh \
  --baseline-ref cde3da1 \
  --refactored-path "$PWD" \
  --backend epoll \
  --benchmark B6-Udp \
  --repeat 3 \
  --output-root "$PWD/build/single-benchmark-triplet-2026-03-15-final/B6-Udp/epoll"
```

单 backend 结果解析：

```bash
python3 scripts/parse_benchmark_triplet.py \
  --input-root "$PWD/build/single-benchmark-triplet-2026-03-15-final/B6-Udp/epoll" \
  --json-out "$PWD/build/single-benchmark-triplet-2026-03-15-final/B6-Udp/epoll/report.json" \
  --markdown-out "$PWD/build/single-benchmark-triplet-2026-03-15-final/B6-Udp/epoll/report.md"
```

## 解释 benchmark 时的注意点

- `B8-MpscChannel` 对 backlog、跨线程 wake 和 batch drain 边界很敏感
- `B14-SchedulerInjectedWakeup` 是 injected fast-path 的 canary
- `B10-Ringbuffer` 与 `B13-Sendfile` 对缓存状态、CPU 频率和单机环境非常敏感
- 网络 benchmark 需要明确 loopback / 容器 / 实机环境
- 文件 benchmark 需要明确缓存状态和文件大小
