# 07-常见问题

## Q1：为什么文档里不再提 `ENABLE_LOG` 或 `spdlog`？

因为当前仓库的真实 CMake 选项只有：

- `BUILD_TESTING`
- `BUILD_BENCHMARKS`
- `BUILD_EXAMPLES`
- `ENABLE_CPP23_MODULES`
- `BUILD_SHARED_LIBS`
- `DISABLE_IOURING`

测试日志宏来自 `test/StdoutLog.h`，不是 `spdlog`。

## Q2：为什么我开启 `-DENABLE_CPP23_MODULES=ON` 后还是没有 import target？

因为 `ENABLE_CPP23_MODULES=ON` 只是“请求开启”，不是“保证开启”。当前项目还要求：

- `CMake >= 3.28`
- 生成器支持模块
- 编译器支持模块

本轮在 AppleClang 17 + Unix Makefiles 上验证时，CMake 明确把模块自动关闭，因此 `E6-MpscChannelImport` 等 target 不存在。

## Q3：为什么 `FileWatcher watcher(scheduler)` 编译不过？

因为真实 API 是：

```cpp
FileWatcher watcher;
auto wd = watcher.addWatch(path, FileWatchEvent::Modify);
auto ev = co_await watcher.watch();
```

构造函数无参数。

## Q4：为什么 `channel.close()` / `co_await channel.send(...)` 编译不过？

当前 `MpscChannel<T>` 的真实接口是：

```cpp
bool send(T&& value);
MpscRecvAwaitable<T> recv();
```

也就是说：

- 发送端是同步 `bool send(...)`
- 接收端才是 `co_await channel.recv()`
- 当前没有 `close()`

## Q5：为什么 `handleTcpNoDelay()` 不存在？

因为 `HandleOption` 当前只公开：

- `handleBlock()`
- `handleNonBlock()`
- `handleReuseAddr()`
- `handleReusePort()`

## Q6：`TimerScheduler` 是最小堆吗？

不是。当前实现基于 `ThreadSafeTimerManager` 的线程安全分层时间轮。

相关源码：

- `galay-kernel/kernel/TimerScheduler.h`
- `galay-kernel/common/ThreadSafeTimerManager.hpp`

## Q7：Linux 上文件异步 IO 需要注意什么？

- epoll 模式下的 `AioFile` 依赖 `libaio`
- `AioFile` 的 `O_DIRECT` 读写要求对齐缓冲区
- 可以使用 `AioFile::allocAlignedBuffer(...)`

## Q10：企业项目要怎么验证 `find_package(galay-kernel)` 真能消费？

先看：

- `docs/00-快速开始.md`
- `docs/02-API参考.md`

当前安装后的 package 会导出：

- `galay-kernel::galay-kernel`
- `galay-kernel::galay-kernel-modules`（仅当模块 target 真实生成并安装）
- `GALAY_KERNEL_VERSION`
- `GALAY_KERNEL_BACKEND`
- `GALAY_KERNEL_INCLUDE_DIR`

当前工作树未包含受版本控制的 package consumer fixture，也未额外导出 `GALAY_KERNEL_SUPPORTED_HEADERS` / `GALAY_KERNEL_INTERNAL_HEADERS` / `GALAY_KERNEL_PACKAGE_CONSUMER_FIXTURE_DIR`。如果要做企业交付 smoke 验证，请单独准备最小 consumer 工程，至少验证：

- `find_package(galay-kernel CONFIG REQUIRED)`
- `target_link_libraries(... galay-kernel::galay-kernel)`

企业交付、`find_package` 与 CI 验证统一看 `docs/00-快速开始.md`、`docs/02-API参考.md`、`docs/07-常见问题.md` 当前章节。
