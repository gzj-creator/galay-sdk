## 本页回答什么

- `MpscChannel<T>`、`UnsafeChannel<T>`、`AsyncMutex` 在体系中的角色是什么
- 哪些并发问题属于主干页应回答的范围
- 哪些测试 / benchmark 是最可靠锚点

## 当前稳定事实

- `MpscChannel<T>` 的发送端是同步 `bool send(...)`，接收端才是 `co_await recv()`
- `UnsafeChannel<T>` 是显式的非线程安全通道，只适合同线程 / 受控调度上下文
- `AsyncMutex` 是协程友好互斥锁；更细的 API 与误用边界已折回主干页
