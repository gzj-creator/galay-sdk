# 10-调度器

本页现在只保留 `Scheduler` / `IOScheduler` 体系的补充定位；完整事实请优先回到主干页。

## 本页回答什么

- `Scheduler`、`IOScheduler`、平台后端类的关系是什么
- 哪些平台 / 宏会影响调度器实现
- 去哪里看公开接口与运行时接入方式

## 当前稳定事实

- 平台后端由构建配置自动选择：Linux 为 epoll / io_uring，macOS 为 kqueue；Windows 当前仍为 planned，不提供可用后端
- `IOScheduler` 负责 IO 相关等待体；`ComputeScheduler` 负责计算协程，两者边界在主干页已有完整说明
- 具体后端差异应以头文件、实现文件和主干页说明为准，而不是本页独立叙述
