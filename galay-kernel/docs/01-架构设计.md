# 01-架构设计

## 仓库结构

- `galay-kernel/async/`：网络、文件、文件监控等异步 IO 类型
- `galay-kernel/kernel/`：任务、调度器、运行时、等待体、定时器
- `galay-kernel/concurrency/`：任务友好同步原语与 channel
- `galay-kernel/common/`：错误类型、时间轮、句柄选项、基础工具
- `examples/`：公开示例
- `test/`：行为验证与回归测试
- `benchmark/`：性能基准

## 导出架构

- 核心库：`galay-kernel`
- 条件模块门面：`galay-kernel-modules`
  - 源文件：`galay-kernel/module/galay.kernel.cppm`
  - 只有模块条件满足时才导出

## 调度模型

### IO 调度

- 平台后端由 `cmake/option.cmake` 自动选择
- Linux：epoll 或 io_uring
- macOS：kqueue
- Windows：planned，当前不提供可配置的后端实现
- `IOScheduler` 负责 socket、文件、文件监控、sendfile 等等待体唤醒

### 计算调度

- `ComputeScheduler` 负责纯计算任务调度
- 公开接口：`start()`、`stop()`、`schedule(...)`、`scheduleDeferred(...)`、`scheduleImmediately(...)`
- 可通过 `Runtime` 统一管理多个计算调度器

### 运行时

- `Runtime` 封装多个 IO / 计算调度器
- `start()` 时自动创建默认调度器并启动全局 `TimerScheduler`
- `stop()` 时统一停机
- 负载分配通过 `getNextIOScheduler()` / `getNextComputeScheduler()` 轮询完成

## Task 与等待体

- 基本任务类型：`Task<T>`
- 链式根任务：`task.then(nextTask)`
- 等待子任务：`co_await childTask()`
- 睡眠：`co_await sleep(duration)`
- Async IO / channel / mutex 都以 awaitable 暴露给任务

## 时间系统

- `TimerScheduler` 使用 `ThreadSafeTimerManager`
- `ThreadSafeTimerManager` 是线程安全分层时间轮，不是最小堆
- 真实实现位于 `galay-kernel/common/ThreadSafeTimerManager.hpp`
- 定时器添加是线程安全的；推进 `tick()` 由单独定时线程完成

## 并发模型

- `AsyncMutex`：协程友好互斥锁，竞争时挂起协程
- `MpscChannel<T>`：多生产者、单消费者；发送端同步 `bool send(...)`，接收端 `co_await recv()`
- `UnsafeChannel<T>`：更轻量的单线程 / 受控场景 channel
- `AsyncWaiter<T>`：一次性结果同步

## 设计取舍

- 命名模块是实验性能力，当前默认关闭
- import 示例的存在不等于当前环境一定会生成 import target
- benchmark 数字受平台、缓存、环回网络与构建类型影响很大
- 文档以真实 target 和真实头文件为准，不再保留旧编号与旧 API 别名
