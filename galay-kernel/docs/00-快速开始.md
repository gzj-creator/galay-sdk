# 00-快速开始

## 适用范围

本页只描述当前仓库已经存在并可验证的构建、安装、运行与最小验收路径。

## 前置条件

- 编译器：支持 C++23
- CMake：`>= 3.16`
- 第三方头文件：需要能找到 `<concurrentqueue/moodycamel/*.h>`
- Linux：
  - `DISABLE_IOURING=ON` 时使用 epoll；若要异步文件 IO，需要 `libaio`
  - `DISABLE_IOURING=OFF` 且系统存在 `liburing` 时才会切到 io_uring
- 命名模块：
  - 仅在 `CMake >= 3.28`
  - 且生成器支持模块（本仓库当前接受 Ninja / Visual Studio）
  - 且编译器不是 AppleClang

## 默认构建

```bash
cmake -S . -B build
cmake --build build --parallel
```

## 开发构建（含测试）

```bash
cmake -S . -B build -DBUILD_TESTING=ON -DBUILD_EXAMPLES=ON -DBUILD_BENCHMARKS=ON
cmake --build build --parallel
```

## 当前安装与消费现状

最小安装命令：

```bash
cmake --install build-docverify --prefix "$PWD/build-docverify/install"
```

当前工作树中的包配置文件会导出：

- `galay-kernel::galay-kernel`
- `galay-kernel::galay-kernel-modules`（仅当模块 target 真实生成并安装）
- `GALAY_KERNEL_VERSION`
- `GALAY_KERNEL_BACKEND`
- `GALAY_KERNEL_INCLUDE_DIR`

说明：

- 当前工作树未保留受版本控制的 `find_package` consumer fixture
- 当前安装包也未导出 `GALAY_KERNEL_SUPPORTED_HEADERS` / `GALAY_KERNEL_INTERNAL_HEADERS` / `GALAY_KERNEL_PACKAGE_CONSUMER_FIXTURE_DIR`
- 如需做安装 smoke 验证，请在独立的最小 consumer 工程中至少验证 `find_package(galay-kernel CONFIG REQUIRED)` 与 `target_link_libraries(... galay-kernel::galay-kernel)`

安装消费、`find_package` 与 CI 验证统一看 `docs/00-快速开始.md`、`docs/02-API参考.md`、`docs/07-常见问题.md`。

## 当前已知限制

- 仓库当前没有 `ENABLE_LOG` 选项，也没有 `spdlog` 依赖链路
- `MpscChannel<T>` 没有 `close()`；发送端是同步 `bool send(...)`
- `HandleOption` 当前只公开 `handleBlock()`、`handleNonBlock()`、`handleReuseAddr()`、`handleReusePort()`
- `AioFile` 只在 `USE_EPOLL` 下公开
- benchmark 数字是当前机器单次运行结果，不是跨平台性能承诺
